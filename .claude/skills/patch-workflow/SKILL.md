# Patch Workflow (Global)

## Алгоритм
1) Сформулируй цель патча одной фразой.
2) PLAN (3–7 пунктов):
   - файлы
   - риски
   - проверки
3) Реализуй изменения минимальным диффом.
4) Проверь:
   - git diff --stat
   - smoke-check проекта (если есть)
5) Сформируй отчёт:
   - изменённые файлы
   - что сделано
   - как проверено
6) Предложи commit message вида: PATCH-XX: <кратко>.

## Запрещено
- расширять объём работ без запроса
- делать “массовый рефакторинг”
- менять форматы данных без явного указания миграции/совместимости
