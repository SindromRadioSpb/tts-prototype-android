# SQLite Migration Safety (Project)

## Когда триггерится
Любые изменения в:
- migrations/*.sql
- db/migrate*.js
- db/sqlite.js
- db/*Repo.js

## Правила
1) Схема меняется только через новый файл migrations/NNN_*.sql.
2) Миграции должны быть:
   - идемпотентны насколько возможно
   - без “ручных правок” в data/app.db в git
3) Проверка:
   - запусти миграции на копии БД или на пустой БД (если поддерживается bootstrap)
   - запусти node tools/step8_2-db-check.js (если он используется как инвариант)
4) Никогда не меняй порядок существующих миграций.
5) Для производительности: индексы добавляй отдельными шагами и фиксируй в миграции.

## Выход
- список изменённых файлов
- что добавлено в схему
- как проверено (команды + результат)
- риск-оценка (низкий/средний/высокий)
