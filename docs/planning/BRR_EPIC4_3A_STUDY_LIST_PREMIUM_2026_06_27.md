## Проблема (владелец 2026-06-27, измерено)
«📚 Учить» v1 показывает **top-12 по частоте по всему тексту** → на большом тексте видна крошечная доля фронтира. Владелец на «מכתבים אחרים» (796 строк) заметил «отображаются не все слова».

**Measure (Kapture, live, профиль 7026, «הכינור» 1769 строк):**
- 8 945 слов-токенов (he) · **973 различных новых слова** (фронтир) · показывалось **12 (1.2%)**.
- Частоты: 466 hapax (1×) / 264 (2–3×) / 243 (4+×). 12-е слово отсекалось на частоте 29.
- Полный скан **1.1 с** (≈0.13 мс/слово) → производительность НЕ ограничение; потолок чисто продуктовый.
- **Качество ранжирования:** топ по частоте в прозе/пьесе = ИМЕНА персонажей, мис-распознанные как контент: `יְהוּדִית` 225×→«еврейский» (имя Йехудит), `צְבִי` 93×→«газель» (Цви), `עַלִּיזָה` 47×→«весёлый» (Ализа). Гомограф-имена офлайн «протекают» (отложены на Tier-3 в gazetteer).

## Ролевой разбор
- **R2 методист:** частота-первой верна; 12 — не сессия и не потолок. Нужна прогрессия порциями; hapax (половина!) ценнее учить по тапу в контексте, чем карточкой.
- **R5 рынок:** LingQ/Readlang показывают ВЕСЬ набор слов урока с сортировкой/фильтром/пагинацией, не cap. Офлайн-скан дешёвый → масштабируемо без сервера.
- **R8 graded:** ценность слова зависит от позиции чтения → scope «весь текст» (подготовка) vs «дальше по тексту» (just-in-time). Приоритет по частотным полосам.
- **R10 морфолог:** чистая частота переоценивает имена; список «учить» с неправильным глоссом имени — харм. Нужен сигнал «возможно имя» / лёгкий ignore / опц. Tier-3.
- **R4 UX:** жёсткий cap без «ещё» = тупик. Показать общее число, порции, сорт/фильтр.
- **R6/R7 куратор/литвед:** в прозе имена доминируют частоту → отделить имена собственные.
- **R11 do-no-harm:** инвариант ключа (collect==paint==save через `statusKeyForCard`) держать; name-flag — СОФТ-подсказка, НЕ меняет резолвер/окраску (отдельный NAME_HINT, не NAME_PROPER); чанковый скан на огромных текстах.

## Решение — ЕДИНАЯ премиум-поверхность «📚 Учить» (A+B+C+D)
Все 4 сходятся в один словарь-вид фронтира:
- **A — полный фронтир + счётчик + порции.** Показать «N из M», рендер чанками (нач. 20 + «Показать ещё»), отмеченные (l1+/known/ignore) выпадают на пересборке.
- **B — scope-переключатель** «Весь текст» / «Дальше по тексту» (rowFrom = текущая верхняя строка чтения → до конца). just-in-time graded.
- **C — качество:** частотные полосы (все / 4+ / 2–3 / редкие) как фильтр; SOFT-флаг «возможно имя» (curated NAME_HINT homograph-имена) + фильтр «скрыть имена» (default off → 0 харма); bulk «отметить видимые → игнор/знаю».
- **D — сорт/фильтр:** сорт частота(деф)/алфавит; фильтры полосы+имена; bulk-действия. (Полный «manage all vocab» вид со статусами кроме фронтира — отложен; «Учить»=фронтир new/undef.)

## Архитектура
**`reader-morph.collectNewWords(mount, statesMap, opts)` v2:**
- `opts`: `{ topN, rowFrom, rowTo }`. Без topN (или большой) → ВЕСЬ фронтир (caller пагинирует). `rowFrom/rowTo` → фильтр спанов по `tr[data-row-idx]` (scope B).
- Возврат +`nameSuspect:bool` (membership в curated `NAME_HINT` — реальные гомограф-имена; SOFT, R1-honest, НЕ меняет resolveCore/окраску). Ключ — через общий `statusKeyForCard` (parity).
- `NAME_HINT` — отдельный набор (≠ `NAME_PROPER`, который суппрессит морфологию): только флаг в списке «Учить».

**`library-ui` — словарь-лист:**
- Header: заголовок + «N из M» (счётчик отфильтрованного/всего).
- Контролы: scope-сегмент (B) · сорт-селект (D) · полоса-фильтр (C) · чекбокс «скрыть имена» (C).
- Лист: chunked render (A) 20 + «Показать ещё»; строка = огласовка + глосс + корень·×freq + бейдж «возможно имя» + статус-селектор `.rm-status`.
- Bulk (D): «видимые → 🚫 игнор» / «→ ✓ знаю» (для быстрой прополки имён).
- one-tap статус → setWordStatus → invalidate readerWordStates + applyDecorations + пересборка списка.
