# Decision Brief — Читальный зал на корпусе Бен-Йехуда

> **Дата:** 2026-06-08 · 1-страничный бриф для владельца. Подробности: gap-матрица

> (`docs/planning/BEN_YEHUDA_READING_ROOM_REQUIREMENTS_BACKLOG.md`), аудит конкурентов
> (`docs/research/COMPETITOR_READING_PRODUCTS_AUDIT.md`), стратегия
> (`docs/strategy/BEN_YEHUDA_LIBRARY_READING_ROOM_STRATEGY.md`).

## Суть в одном абзаце
Наш **движок чтения и обучения сильнее всех пяти конкурентов по Hebrew-глубине**
(морфология на тапе, Pealim 9279 офлайн, i+1 frontier, двойная транслитерация,
Anki-sync, конкорданс) и производство (bundle ZIP v2, MT-кэш, BYOK, enrich-script)
зрелое. **Проигрываем в одном: библиотека/discovery и сама чистая поверхность Зала —
их ещё нет.** Стратегия D («Издательство + Читальный зал») уже закрывает это; задача —
построить чистый Зал поверх готового движка, наполнить его курируемым двуязычным+аудио
каноном и честно показать провенанс. Это даёт продукт, которого **нет ни у кого**:
подлинный ивритский литературный канон, проходимый на любом уровне, с native-аудио,
морфологией на тапе, офлайн.

## Top-10 пробелов
1. Нет чистой страницы Зала (`library.html`). **P0**
2. Нет полок/коллекций + редакторской курации (discovery = плоский список). **P0**
3. Нет модели метаданных корпуса (автор/эпоха/жанр/переводчик/язык-ориг + content-hash). **P0**
4. Нет конвейера ингестии Бен-Йехуда → бандлы. **P0**
5. Нет провенанса/атрибуции + меток «вычитано/машинно/TTS». **P0**
6. Нет консоли скаффолдинга (никуд-fade/транслит/reveal перевода) — данные есть. **P1**
7. Сложность/покрытие и «следующий для тебя» не показаны (i+1 есть в данных). **P1**
8. Нет audio↔text karaoke-подсветки. **P1**
9. Нет статуса слов (known/learning/new) в чтении + лёгкого захвата. **P1**
10. Ноль cost/quota-прозрачности BYOK + нет PWA/офлайн-cues + тонкий error-recovery. **P1**

## Top-10 практик перенять (адаптируя, не копируя)
1. **Sefaria:** кураторские Collections + редакторский голос → наши полки/маршруты.
2. **Sefaria:** провенанс/атрибуция per work + free/open + офлайн-first (= наш OPFS).
3. **Beelinguapp:** параллель side-by-side как основной режим + karaoke audio-sync.
4. **LingQ:** corpus-wide статус слова, видимый прямо в чтении (данные у нас есть).
5. **LingQ/StoryHebrew:** уровень-бейджи + «next for you» → наш i+1 как рекомендация.
6. **StoryHebrew:** регулируемый никуд full/partial/off (+ наша транслитерация).
7. **Readlang:** frictionless «tap→перевод→карточка» + контекстные объяснения.
8. **Beelinguapp/Sefaria:** многонаправленные пары (Heb↔Ru, Heb↔En) + офлайн-загрузка.
9. **Sefaria:** developer-API/шаринг (позже) как платформенный ход.
10. **Все:** лёгкий onboarding/first-run + честный free-tier с реальной пользой.

## Top-5 практик отклонить
1. **AI-генерация «graded» рассказов** (Beelinguapp) — подлинный канон = моат.
2. **Агрессивные lookup-paywalls/лимиты** (LingQ) — public-domain бесплатно, монетизация Студии/BYOK.
3. **Браузерный TTS как основной / метка «native» для TTS** (Readlang/Beelinguapp) — честность R1.
4. **Infinite-scroll memory-heavy читалка** (Sefaria) — пагинация/lazy + главы.
5. **10 конкурирующих переводов** (Sefaria) — 1–2 помеченных параллельных.

## Рекомендуемая форма MVP (P0-набор)
Чистая `library.html` (reuse `renderTable`/`v3LibraryOpenText`/тема/OPFS/i18n; скрыты
все Студия-функции) · два трека полок (Доступная/Литературная), засеянные ~50–150
вручную вычитанными двуязычными+аудио произведениями (shipped офлайн-бандлы) · модель
метаданных корпуса (schema-first) · провенанс + атрибуция + честные метки · бейдж
сложности + простой «следующий для тебя». Студия остаётся мастерской (bulk-импорт,
MT/морфо-override, Anki/SRS, граф, BYOK-обогащение глубоких стеллажей).

## Главный риск
**Зал превращается в «index.html lite»** (теряет чистоту) **и/или сваливание
неградуированного архаичного 20K-корпуса** (overwhelm + регистр-mismatch).
*Митигация (встроена в P0):* жёсткое разделение Студия↔Зал · кураторский two-track
on-ramp · честные метки регистра/эпохи · глубокие стеллажи — позади курации.

## Самая быстрая премиум-победа
**Консоль скаффолдинга** (никуд-fade + раскрытие перевода) **+ audio-sync подсветка** —
данные уже есть, это чистый UX, мгновенно отстраивает от StoryHebrew/Beelinguapp.

## Лучшая моат-фича
**Персональный i+1 «следующий для тебя» по подлинному литературному канону с
морфологией-на-тапе** — для иврита этого нет ни у кого (LingQ грубо, Sefaria нет,
StoryHebrew вручную). Усилено офлайн-first + провенанс-честностью.

## Рекомендация: что строить первым и почему
Идти строго **schema-first → поверхность → контент → честность**, затем скаффолдинг:
1. **BRR-P0-001** метаданные корпуса (иначе болезненная миграция позже).
2. **BRR-P0-003 + BRR-P0-002** полки → чистая страница (поверхность нужна непустой).
3. **BRR-P0-004** конвейер ингестии → ~50–150 curated произведений (2 трека).
4. **BRR-P0-005** провенанс + честные метки (доверие = фича, R1/R5 invariant).
5. Затем **BRR-P1-006/007/008** (скаффолдинг + i+1 + audio-sync) — премиум-ощущение и моат.

Это даёт публично-пригодную бету Зала, которая сразу превосходит конкурентов там, где
у нас уже есть движок, и закрывает единственный реальный пробел — discovery/чистую поверхность.
