## ✅ Что подтверждено живым (попутно)
Скриншот — это **live-верификация W3+W4** на реальном профиле: бэнд «легче/средне/сложнее» + «много имён/архаики» (W3) и карточное покрытие «≈N%» (W4) рендерятся корректно. То есть OPFS-лок снялся, профиль читается. Полиш-пробелы — это следствие сосуществования всех сигналов на узкой (132px) карточке, а не дефект W3/W4.

## Корни (консолидировано, заземлено)
1. **Инверсия иерархии + цветовая коллизия.** Самый «громкий» элемент карточки — наименее полезный провенанс: `«Машинный перевод»` рендерится ПЕРВЫМ и безусловно (library-ui.js:280-283), залит насыщенным amber `#fef3c7` (library.html:166). А учебные сигналы (W3-бэнд, W4-покрытие) добавляются позже, лениво. **Тот же amber = и «Машинный перевод» (провенанс), и «средне» (сложность)** (library.html:166 ≡ :207). Green переиспользован на 4 осях (rs-human/audio-human/coverage-in/diff-easy). Цвет кодирует «хорошо/плохо», а не «какой сигнал».
2. **Шумовые бейджи-инварианты.** В каталоге **796/796** работ `review_status='machine'` и **796/796** `audio_status='none'`. → `«Машинный перевод»` и `«Без озвучки»` константны на всех карточках; `«Без озвучки»` рекламирует ОТСУТСТВИЕ фичи (corpusProvBadge('audio','none'), library-ui.js:284-287).
3. **Голый «≈34%».** Подпись только в `title`-тултипе (library-ui.js:3048-3049) — на тач-мобайле невидим. При этом ключ `room.corpus.cov.familiar`=«знакомо» **уже есть и НЕ используется** на карточке (ru.js:2165).
4. **Утечка редакторских скобок в заголовок.** Заголовок рендерится сырым (`card.title`, library-ui.js:268-269), helper-очистки нет. В каталоге **83/796** title содержат `[`, **35** — вариант-ноту `[נוסח 'שירים']`, **51** — целиком в скобках (incipit). ⚠ **do-no-harm:** скобка иногда ЕДИНСТВЕННЫЙ дизамбигуатор (id 36 `לאיס"י [הבבלי]` vs id 42 `לאיס"י [איש חמודות]`); трейлинг `«…»` (48 работ) — легитимный incipit, НЕ резать; гершаим `לאיס"י` — НЕ трогать.
5. **Внутренний ритм + CLS.** Карточки РАВНОВЫСОКИ (`.shelf-rail` stretch). Реальный дефект: `.work-card{justify-content:space-between}` (library.html:141) распихивает слабину при коротком заголовке → бейдж-ряды плавают (замер meta-top = 90/89/134px); ленивые W3/W4-строки добавляются после пейнта (IntersectionObserver, library-ui.js:3028/3048/3087) → **карточки растут, рейл пере-растягивается = layout-shift**; одна карточка с «архаики» инфлирует ВЕСЬ рейл.
6. **Слабый/двусмысленный CTA + нет focus.** Вся карточка `role=button` (library-ui.js:267), но «Читать»/«Открыть» — бледный 12px `<span>` (library.html:152) = расщеплённая tap-модель; нет `:focus-visible`/`:hover` (есть у `.rm-w`, `.continue-done` и др. — карточки пропущены).
7. **Жаргон в копирайте секции.** `«Машинный перевод (Gemini) + Dicta-никуд. Провенанс на каждой карточке.»` (ru.js:2155) — инсайдерские термины для учащегося.

## Требования (приоритизировано · критерий приёмки · фикс · роли · усилие)

> Приоритет: **P0** ломает премиум-восприятие/честность · **P1** премиум-пробел/честность · **P2** консистентность/полиш · **P3** nice-to-have. Усилие S(часы)/M(1–2д)/L(>2д). Все Room-only, `smoke:reader-parity` зелёный, @380px свет+тёмная перед коммитом.

### PC-1 · Когерентная семантика бейджей: один цвет = одна ось · **P0 / M · R4+R9+a11y+visual**
**Проблема:** amber кодирует и провенанс, и сложность; green — 4 оси; цвет = «хорошо/плохо», не «какой сигнал» (library.html:166≡207; 168≡171≡187≡206). На карточке до 5 пилюль, шумовая — самая яркая.
**Требование:** провенанс / личное-покрытие / intrinsic-сложность РАЗЛИЧИМЫ не только цветовой рампой (ведущий глиф/иконка ИЛИ форма ИЛИ выделенная строка + подпись). Один fill-цвет = ровно одна семантическая ось. На карточке ≤3 пилюли одновременно. **Гейт:** computed `background-color` `.diff-band.diff-mid` ≠ `.prov-badge.rs-machine`.
**Фикс:** провенанс — единственное «залитое» семейство; difficulty — outline/ghost-пилюля с ramp-глифом (`●○○`) на своей строке (W3 уже даёт `.work-card-difficulty` + `border-radius:8px` ≠ pill 999px — усилить это); coverage — с подписью (PC-3). Документировать карту ось→цвет рядом с CSS (library.html:155-219).

### PC-2 · Подавить инвариантные провенанс-пилюли (убрать рекламу отсутствия) · **P1 / S · R5+R6+R9**
**Проблема:** `«Без озвучки»` (796/796 none) и `«Машинный перевод»` (796/796 machine) — нулевая различающая информация; «Без озвучки» = реклама отсутствия (library-ui.js:284-287). Заголовок секции уже это сообщает.
**Требование:** аудио-пилюля рендерится ТОЛЬКО при `audio_status∈{tts,human}`; rs-пилюля на рейле — только при non-default (`machine_assisted`/`human_proofread`); инвариант — в заголовке секции, не 796×. **Гейт:** none-audio карточка не содержит узла `.prov-badge.audio-none`.
**Фикс:** гард `if (au!=='none')` (library-ui.js:284-287) + `if (rs!=='machine')` на рейле; зеркалить в work-row (library-ui.js:4634). Освобождает слоты для учебных сигналов (с PC-1).

### PC-3 · Подписать покрытие на карточке (не тултип) · **P1 / S · R4+R6+R9+a11y**
**Проблема:** голый «≈34%», подпись только в `title` (невидим на тач); ключ «знакомо» уже есть, но не используется (library-ui.js:3048; ru.js:2165).
**Требование:** покрытие несёт ВИДИМУЮ микро-подпись («≈34% знакомо» / глиф+%), визуально отделено от intrinsic-бэнда (разные оси). Accessible-name раскрывает смысл, не голый процент.
**Фикс:** `text:'≈'+pct+'% '+tt('room.corpus.cov.familiar')` (library-ui.js:3048), `title` оставить полным; сгруппировать с difficulty в учебный кластер «о тебе / о тексте», отдельно от провенанса. Проверить перенос на 132px.

### PC-4 · Чистый заголовок: убрать редакторские скобки для показа · **P1 / M · R4+R6+R7+R9**
**Проблема:** сырой `card.title` тащит `[נוסח 'שירים']` в заголовок (library-ui.js:268-269); 83/796 со скобками.
**Требование:** карточный заголовок показывает ЧИСТЫЙ title; **трейлинг** редакторская скоб-/скобочная вариант-нота не в заголовке (вынести в muted-подстроку — НЕ выбрасывать: дизамбигуирует). Легитимные incipit-`«…»` и гершаим — НЕТРОНУТЫ. **Гейт:** ни один рендер-заголовок не содержит подстроку `«[נוסח»`.
**Фикс:** PURE `corpusTitleParts(raw)→{title,note}` (снять ТОЛЬКО финальную `/\s*[\[(][^\])]*נוסח[^\])]*[\])]\s*$/` + общий финальный дизамбиг-`[…]`; `^\[(.+)\]$` → title=inner + тег «загл. по 1-й строке»; трейлинг `…` не трогать). Применить в renderCorpusCard:268 + renderCorpusWorkRow:4614; `note` — стилем `.work-card-author`. **Лучше:** эмитить `title_clean`+`title_note` из `build-corpus-catalog.js` (поиск сохраняет полное), client-helper — фолбэк. Добавить ассерт в corpus-smoke.

**Проблема:** `justify-content:space-between` (library.html:141) → плавающие бейдж-ряды; ленивые W3/W4-строки → CLS; «архаики»-строка инфлирует рейл.

**Фикс:** `.work-card{justify-content:flex-start}` + `.work-card-cta{margin-top:auto}` (library.html:141,152); `.work-card-title{min-height}` под clamp **2** строки (library.html:148: 3→2); зарезервировать высоту difficulty-строки (рендерить контейнер всегда / инлайнить чип в meta) — ленивый append ЗАПОЛНЯЕТ, не растит.

## Рекомендованная имплементация (один полиш-эпик, по волнам)
- **Волна 1 (честность сигналов — наибольший премиум-эффект):** PC-2 (де-шум) → PC-3 (подписать покрытие) → PC-1 (цвет=ось) → PC-7 (контраст). Это убирает 2 шумовых пилюли, делает учебные сигналы доминантой и читаемыми — карточка сразу «дышит».
- **Волна 2 (геометрия + заголовок):** PC-5 (ритм+CLS, clamp 2) → PC-4 (чистый title). Карточка перестаёт «плавать»/рефлоу-ить.
- **Волна 3 (взаимодействие + копирайт):** PC-8 (tap+focus) → PC-6 (копирайт) → PC-9 (автор).
- **Волна 4 (полировка):** PC-10 (поэзия) · PC-11 (длина) · PC-12/13/14/15.
**Каждая волна:** гейты (corpus-vocab/reader-parity/i18n + новый title-clean/contrast-ассерт) → @380px свет+тёмная (+ before/after на CLS) → деплой → prod-verify. **Перед кодом — adversarial-review субагентом** (как в Эпике 5: ловил CRITICAL).

## 🔀 Открытые дизайн-развилки (нужно решение владельца перед кодом)
