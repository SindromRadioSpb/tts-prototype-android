## Что это и зачем

причина слабого тап-резолва — **контентные слова без огласовки** (32% нерезолва). Корень: оригинальный
бейк оставил `hebrew_niqqud` ПУСТЫМ/частичным у 110 работ (`audit:corpus-niqqud` → `reniqqud-ids.json`).
Их `hebrew_plain` + перевод целы — не хватает только огласовки. Dicta-Nakdan вокализует plain-текст ~98%.

## Инструмент (новый)
`scripts/premium/reniqqud-fill.js` (`npm run reniqqud:fill`) — **niqqud-only** перебейк, на месте:
- заполняет ТОЛЬКО `hebrew_niqqud` из Dicta-cloud (keyless); перевод/аудио/порядок — НЕ трогает;
- **R1 make-or-break:** пер-словная реконструкция (`reconstructNiqqud`) — слову подставляется огласовка,
  ТОЛЬКО если консонантный скелет совпадает с Dicta; ктив-нормализованные слова остаются БЕЗ огласовки
  → **скелет исходного текста НЕ меняется никогда** (гейт `npm run smoke:reniqqud`, 5 кейсов);
- crash-safe леджер `.tmp/benyehuda/reniqqud-fill-ledger.json`, резюм = та же команда;
- per-work before/after; флаг works без лифта; пишет `_reniqqud` маркер в работу.
- Флаги: `--dry-run` (замер) · `--bake` · `--status` · `--limit N` · `--ids-file` · `--min-cov 60` · `--row-cov 50`.
