# Эпик B «Мои тексты» в Читальном зале · 2026-07-02

**Статус:** реализация одобрена владельцем (утверждённая последовательность — recon context-overlay §9, п.2).
Роли: R4 (вед.) · R5 · R2 · R11.

## Решение по UI/UX (вопрос владельца: переиспользовать окно «Библиотека (v3)» Студии?)

**НЕТ — переиспользуем паттерны Зала, не окно Студии.** Причины:
1. **Технически компонент не существует:** живой JS Студии — inline в `index.html` (39K строк,
   ловушка `feedback_studio_live_source_inline`); «Библиотека (v3)» — не модуль. Извлечение =
   вскрытие index.html (запрещено parity-инвариантом до Stage 2) либо iframe-костыль.
2. **UX-язык:** окно v3 — консоль управления power-user'а (12+ кнопок, Изменить/Обогатить/
   Удалить/Сбросить). Зал — премиальная читальная поверхность. Премиально = НЕ приносить пульт
   управления в читальню; полка в родном языке Зала (карточки/прогресс/Продолжить/поиск),
   управление — по ссылке в Студию.
3. **Ценность v3 переносится паттернами:** поиск/сортировка/прогресс/Продолжить — у Зала уже
   есть свои компоненты (Continue-полка, corpus-фильтры, `.work-card`, `.corpus-search-input`).

## Архитектура (дёшево: хранилище УЖЕ общее)

- Один OPFS `localDb` на обе поверхности; корпусные работы и студийные тексты — одна таблица
  `texts`. **Дискриминатор канонический:** текст = corpus ⇔ `source_meta_json.corpus` задан
  (`db/premium/corpusMeta.js`). «Мои тексты» = НЕ corpus ∧ НЕ archived.
- Полка на corpus-home (порядок сверху: Continue → **Мои тексты** → Прочитанные → …), rail до 12
  недавних; кнопка «Все (N)» разворачивает in-place: клиентский поиск (title/tags/source/topic)
  + вертикальный список + ссылка «Управлять — в Студии». Один запрос `listTexts(limit 500)`,
  ноль per-item fan-out (урок `feedback_test_with_nonempty_profile`).
- Открытие — тот же byte-parity Room-reader (`openReader(id)`); прогресс — «строка N»
  (честно, без выдуманных %, т.к. n_rows нет в texts).
- **Честные деградации (R11):** оверлеев (context/proclitic) у своих текстов нет → un-baked
  семантика P4-цепочки (live Tier-3 + consent) автоматически; никаких полос сложности/бейджей
  корпуса; никаких management/destructive действий в Зале.

## Гейты

`smoke:reader-parity` (билдер не тронут) · SW bump.
