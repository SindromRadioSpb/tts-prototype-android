# Reading-Room UX requirements — current-row / resume / bookmark / search-hit highlight + read-aloud

> Сформировано 2026-06-15 из исследования: наш **Studio** (`index.html` row-states + `studio-karaoke.js`) +
> конкуренты (Kindle/Whispersync · Apple Books · Readwise · Moon+ · Speechify · NaturalReader · Voice Dream ·
> **LingQ** sentence-mode · **Beelinguapp** parallel-karaoke · **Sefaria** search-in-text · `::target-text`) + WCAG.
> Премиум-планка для билингв-RTL-ридера. Роли R4/R8/R5/R10-a11y/R2. Источник для будущего полиша; не всё реализовано сразу.

## Принцип (главный вывод исследования)
**Стиль зависит от СОСТОЯНИЯ — не один уникальный вид на всё.** Смешение состояний в один вид стирает у пользователя
способность их различать (самая частая ошибка). Палитра состояний:
| Состояние | Персистентность | Канал | Реализовано |
|---|---|---|---|
| **resume / закладка / FTS-прыжок** (jump-target) | до взаимодействия | заливка + ведущая полоса, **амбер** (distinct от playback) | ✅ BRR-P2-005 `.rm-row-jump` |
| **read-aloud текущая строка** | пока играет, движется | заливка строки (**синяя** в Зале) + полоса | ✅ `.row-playing`/`.karaoke-on` |
| **read-aloud текущее слово** | пока играет, бежит | пилюля-заливка (янтарь), не bold/underline | ✅ `.rm-w-speaking` |
| **ручная закладка** (маркер в списке) | постоянный | ★ в action-cell + полка | ✅ BRR-P2-003 |

## a11y / RTL инварианты (load-bearing, WCAG)
- **Не только цвет** (1.4.1): у активного состояния есть второй канал (полоса/маркер) — пережить дальтонизм/grayscale.
- **Non-text contrast ≥3:1** (1.4.11) для полосы/границы-индикатора — в **обеих** темах.
- **Билингв-связка:** подсветка строки красит ОБЕ языковые колонки (он же `td`+`td.rtl`) — иначе теряется параллель-ценность.
- **RTL/никуд:** НЕ подчёркивание под огласованным ивритом (бьёт по te'amim/никуду); полоса логически на ведущем крае;
  заливка не клиппит огласовку @380px.

2. ✅ **Jump-target подсветка** для resume/закладки/FTS, держится до взаимодействия, distinct hue. [BRR-P2-005]
3. ✅ **FTS-хит открывается НА совпавшей строке** + подсветка. [BRR-P2-005]
4. ✅ **Active-row read-aloud подсветка билингв-связкой** (строка) + **бегущее слово**. [karaoke-семейство]
5. **search-hit ≠ read-aloud по оттенку** — у нас jump=амбер, playback=синий ✓ (различимы).
6. ✅ a11y не-только-цвет + RTL-геометрия. [BRR-P2-005]

## P1 — премиум-дифференциаторы (выше LingQ/Beelinguapp) — БЕКЛОГ
7. **Tap-to-activate любой строки** → стать «текущей», проиграть только её + показать перевод (Beelinguapp «tap sentence to replay»
   + Readlang/Migaku selection). Два «current» (audio-driven + tap-driven) одним визуальным языком.
8. **User highlight-mode toggle** (none / слово / строка / оба) + выбор цвета — как Apple/NaturalReader/Voice Dream. Русскоязычным
   читателям огласованного классич. иврита нужна настройка легибельности.
9. **Auto-scroll держит активную строку в комфортном бэнде** (центр) — чтобы не «гнаться» за подсветкой @380px (Voice Dream «Focused»).
10. **FTS snippet-в-результатах** (совпавшая строка прямо в карточке результата, как Sefaria) — сейчас «найдено», но без строки-превью.
11. **«Прочитано/завершить»** для текста в «Продолжить» (убрать со 100% / mark-finished) — Continue-список не копит завершённое.
