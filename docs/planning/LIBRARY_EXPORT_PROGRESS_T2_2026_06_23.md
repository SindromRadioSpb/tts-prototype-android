## 1. Требования (functional)

### R-PRE — Пре-флайт (ожидание под контроль)
- **R-PRE-1** При клике «Экспорт ZIP (с аудио)» НЕ начинать сразу. Показать сводку:
  кол-во текстов (точно), кол-во аудио-ассетов (точно), оценку размера (≈, эвристика),
  оценку времени (≈, калибр-константа).
- **R-PRE-2** Три действия: **[Экспорт с аудио]** · **[Только метаданные (быстро)]** · **[Отмена]**.
- **R-PRE-3** «Только метаданные» = ZIP без папки `audio/` (manifest + library.json + notes_advanced).
  На приёмнике аудио восстановится из content-addressed кэша при первом воспроизведении
  (та же философия, что lightweight text-card). Помечать `partial_backup:true`, `audio_count:0`.
- **R-PRE-4** Порог soft-warn: если аудио > 3000 ИЛИ оценка размера > 200 МБ — выделить
  строку времени/размера (amber) + подсказка «можно «Только метаданные»». Без жёсткого кэпа.

### R-CARD — Прогресс-карточка (вместо одной строки статуса)
- **R-CARD-1** Детерминированный бар. Вес фаз: аудио-фетч 85% + упаковка 15% (константы).
- **R-CARD-2** Степпер: ①Чтение → ②Аудио → ③Упаковка → ④Готово. Состояния active/done(✓)/idle.
- **R-CARD-3** Счётчик + накопленный размер: «Аудио 4328/8906 · 180 МБ».
- **R-CARD-4** ETA + темп (скользящее окно ~5с): «Осталось ~6 мин · 14/сек». Прячется, пока окна нет.
- **R-CARD-5** Живой индикатор пропусков: «⚠ 2» (timeout/404), тултип с разбивкой причин.
- **R-CARD-6** Кнопка **[Отмена]** на всех фазах.
- **R-CARD-7** Кнопка «Экспорт…» в тулбаре: `disabled` + inline-спиннер, пока идёт.

### R-CANCEL — Отмена (вынесено в требование любого уровня; сейчас тупик)
- **R-CANCEL-1** Единый `AbortController` на весь прогон. Отмена: abort всех аудио-фетчей +
  флаг останова цикла + прерывание `generateAsync` (через флаг — JSZip не abort-ится, но мы
  не вызываем generateAsync, если отменено до упаковки; во время упаковки — даём завершить либо
  бросаем после неё, не сохраняя). Никаких частичных файлов на диск.
- **R-CANCEL-2** После отмены: статус «Экспорт отменён», карточка сворачивается, кнопка снова активна.

### R-CHIP — Глобальный чип (переживает закрытие модалки)
- **R-CHIP-1** Если модалка библиотеки закрыта во время экспорта — показать плавающий чип
  (bottom-right, выше safe-area; НЕ bottom-center, чтобы не конфликтовать с PWA-update-тостом):
  «⬇ Экспорт… 58% ✕».
- **R-CHIP-2** Чип кликабелен → открывает модалку обратно на карточке. ✕ = отмена.
- **R-CHIP-3** При завершении/ошибке/отмене чип авто-исчезает (success — короткий «готово»).

### R-I18N / R-A11Y / R-MOBILE
- **R-I18N** Все новые строки — ключи в ru/en/he (гейт `smoke:i18n` симметрия). Шаблоны с {плейсхолдерами}.
- **R-A11Y** Бар `role="progressbar"` + `aria-valuenow/min/max`; live-регион для текста статуса (`aria-live="polite"`).
- **R-MOBILE** Карточка и чип корректны @380px RTL. Кнопки карточки/пре-флайта обязаны иметь
  исключение из `button{width:100%}` (CSS-ловушка #1): `#v3LibExportCard button { width:auto; }`
  (ID-специфичность), для пре-флайт-кнопок — оставить ширину 100% (вертикальный стек — ОК).

## 2. Критерии приёмки (acceptance)

1. Клик по «Экспорт ZIP» → пре-флайт с точными счётчиками и ≈размером/≈временем; три кнопки работают.
2. «Только метаданные» → ZIP без `audio/`, открывается, импортируется; быстрый путь (секунды) —
   используется как **быстрая прод-верификация** (без 360-МБ прогона).
3. «Экспорт с аудио» → карточка: бар растёт 0→100%, степпер переключается, счётчик/размер/ETA идут,
   «⚠N» при пропусках; по завершении — итог + скачивание.
4. **Отмена** на фазе аудио реально останавливает фетчи (видно в Network: новые `/api/audio` прекращаются)
   за ≤1–2 с; файл не скачивается; кнопка снова активна.
5. Закрытие модалки во время прогона → чип внизу; клик возвращает на карточку; ✕ отменяет.
6. @380px RTL: карточка/чип/пре-флайт без переполнения, кнопки не растягиваются криво (скриншот-гейт).
7. Гейты зелёные: `smoke:i18n` (новые ключи ×3), `smoke:reader-parity`, синтаксис; SW bump.
8. Прод-верификация (Node-fetch SW + Kapture @380px на реальном профиле владельца).

## 3. Технический план (фазами, каждая — gates + prod-verify, commit+push по умолчанию)

- ✅ **Фаза 0 — Движок+карточка (T1-ядро):** экспорт вынесен в `v3ExportRunLocal()` (стейт-машина
  + один `AbortController`); DOM-карточка `#v3LibExportCard` + CSS `.v3-exp-*`; бар/степпер/счётчик/размер/Отмена/busy.
- ✅ **Фаза 1 — Пре-флайт + «только метаданные»:** `v3ExportPreflight()`; JSON-only ветка (`export_mode:'meta'`,
  без папки `audio/`). Даёт быстрый verify-путь.
- ✅ **Фаза 2 — ETA/темп + живые пропуски:** `v3ExportTrackRate()` (скользящее 5с-окно), ETA, «⚠N»+тултип.
- ✅ **Фаза 3 — Глобальный чип:** `v3ExportChipPaint()` — плавающий чип (bottom-right / RTL bottom-left),
  возврат в модалку, отмена.

**Где код:** всё в `public/index.html` inline — движок `v3ExportRunLocal` + хелперы `v3Export*` + `v3ExpT`
i18n-фолбэк; CSS `.v3-exp-*`; HTML-хост `#v3LibExportCardHost`. i18n namespace `exp.*` (ru/en/he).

**Файлы:** `public/index.html` (inline JS+HTML+CSS — live Studio source), `public/i18n/locales/{ru,en,he}.js`,
`public/sw.js` (bump). Без сервера. index.html/reader-core — только Studio-часть экспорта (renderTable не трогаем → reader-parity цел).

**Оценка размера (R-PRE-1):** count точно; size ≈ count × avg(~42 КБ, калибр из 360МБ/8904) ИЛИ сумма
известных `size_bytes` из таблицы `audio_assets`, если доступны (в `exportBundle` они null). ETA ≈ remaining/rate,
rate — живой; до старта — грубая константа.

**Решения владельца (приняты 2026-06-23):** (a) «Только метаданные» = без аудио, восстановление на
приёмнике из кэша — ✅ да · (b) чип bottom-right (RTL → bottom-left) — ✅ да · (c) гейт = форма
metadata-only + live Kapture/owner @380px — ✅ да.

## 4. Следующие шаги (после T2 — defer, по слову владельца)

Две независимые семьи требований, комбинируются:

- **T3 — флагман без фризов + поток на диск (~1.5–3 д).** Web Worker offload (UI не дёргается на больших
  упаковках) + File System Access API: выбор файла *сразу*, потоковая запись чанками (ограниченная память —
  нет ~360 МБ blob в RAM, нет внезапного «Сохранить как» в конце). Fallback на текущий blob-путь для iOS/Firefox/Safari.
  Ценность растёт с размером библиотеки (у владельца 8906 клипов).
- **P — перф-трек: ускорить ожидание (~1–2 д).** Не про показ, а про скорость: ↑ параллелизм аудио-фетча +
  чтение аудио прямо из OPFS-кэша вместо сетевого `/api/audio` (8906-кейс: измеренные ~7–16/сек, цель ~10 мин ⇒ ~2 мин).
  Для реального использования даёт больше «премиума», чем любая анимация. **Рекомендация: P → затем T3.**

Прочий смежный мелкий бэклог (опционально): экспорт-поверхность в Зале (`library.html`); более богатые
docs по мобильному импорту; headless-смоук формы metadata-only-бандла как постоянный гейт.
