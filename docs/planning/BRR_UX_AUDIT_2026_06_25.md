## 1. Флагманская находка (живой баг) → почему Эпик 1 = P0
Тап по первому слову текста «הלך נפש»: карточка показывает **הֵלֶךְ**, **корень הלך**, **POS существительное**, но **глосс — предложный** («к … дательным падежом; у … принадлежность; с ש/אצל/־») и стоит бейдж **«точно»**. Это уверенно-ложный глосс на флагманской поверхности (морфология-на-тапе = моат №1).

Код-причина (подтверждена): `formFirstResolve` (`notes-autogen.js`) — единственный резолвер-путь БЕЗ guard'а кратности: на огласованной ячейке-гомографе (`ids.length>1`) берёт `arr.find(pos)||arr[0]` вместо `null`, далее `resolveContentUnit` штампует conf 0.92/`form-first` → `provenanceLabel` безусловно → «exact/точно». Сёстры (`resolveTrueRoot`, `offlineMeaningLookup`) на кратности возвращают `null` — т.е. непоследовательность реальна. **Сильнейший бейдж уверенности тратится на неразрешённые коллизии** — худшее нарушение инварианта владельца «без заглушек, бескомпромиссное качество».

## 2. Эпики (приоритет → последовательность)

### ЭПИК 1 — Честность резолвера: никогда не тратить «точно» на неразрешённое чтение · **P0 / L · R10**
Сделать таксономию уверенности карточки структурно-честной: «точно» только когда офлайн-чтение действительно решающее; все уверенно-ложные пути честно деградируют.
Находки: `morph-content-homograph-exact` (P1 коллизии-гомографы), `morph-proper-name-gate` (P1 — имена собственные фабрикуют корень+ложную «семью корня» = классич. R3 ложный узел), `morph-missing-word-honest` (P1 — слово вне 9279-словаря отдаёт чужой глосс вместо «нет в офлайн-словаре»; для 26K архаики это ЧАСТЫЙ случай, не хвост), `morph-confidence-gates-enrichment` (P2 — гейтить Pealim-direct/таблицу/семью той же уверенностью), `morph-niqqud-context-honesty` (P2 — покрытие никудом бимодально 56.6%), **`morph-exact-precision-gate` (P1 — измерительный харнесс, СТРОИТСЯ ПЕРВЫМ)**.
Последовательность: **сперва харнесс** (`smoke:reader-morph:audit`, precision-floor ≥98% на N≥300 реальных строк vs Dicta-контекст, валит сборку ниже пола) → дешёвая демоция `formKey→Set(pid)` (коллизии) → gate имён → gate обогащения → borrowed-vs-unknown → никуд-бейдж.
Риск: калибровка (over-демоция затопит «подобрано» — мерить recall/FP раздельно, R10-норма). Silver≠gold (Dicta тоже врёт на архаике — тегать доверие).
Успех: gate `smoke:reader-morph:audit` precision ≥98%; ноль фабрикаций имён/borrowed-as-«точно»; ручная @380px проверка הֵלֶךְ/חַיִּים/משה/ירושלים.

## 3. Кросс-режущие инварианты и нормы (применять во ВСЕХ эпиках)
1. **Honest-gate — общая зависимость:** Эпики 2/3/4/5 потребляют сигнал уверенности/коллизии Эпика 1. Обогащение/refine/раскраска/пред-обучение НИКОГДА не богаче, чем разрешение уверенно. **Эпик 1 — первым.**
2. **MEASURE-before-code (R10):** харнесс precision (Эпик 1) — шаблон; любое изменение резолвера/лейблинга гейтится recall/FP на реальных baked-работах, не спот-чеками. Volume-зависимый UI (статус-fan-out, end-of-text пересчёт) — тест на БОЛЬШОМ профиле владельца, не свежем OPFS.
3. **Provenance везде, derived≠asserted≠curated (R4/R9):** каждая новая поверхность (бэнды сложности, даты/эпоха автора, машинный никуд, moat-copy, интро) несёт честный провенанс и честное «unknown».
4. **Инвариант-безопасность единообразна:** всё Room-only; `index.html` + parity-locked билдер нетронуты; `smoke:reader-parity` зелёный; offline-first/OPFS. Десктоп-max-width НЕ в общий CSS.
5. **Mobile-first RTL @380px + новый десктоп-слой:** скрин @380px (светлая+тёмная) перед коммитом; Эпик 7 добавляет десктоп-брейкпоинты — проверять ОБА.
6. **WCAG / не-только-цвет — стоящий инвариант** (команда уже задокументировала): статус слов, prov-pills, reduced-motion, read-aloud честные состояния — в ОБЕИХ темах + второй (не-цветовой) канал.
7. **Удержание — стратегическая нить (R2/R5):** Эпики 3/4/5 вместе = цикл learn→recall→next-step (моат LingQ/graded), БЕЗ второго SRS-планировщика (Anki держит расписание).
8. **Локализация:** новые строки — сразу в ru/en/he (гейт `smoke:i18n`); HE-локаль требует RTL-корректной РАСКЛАДКИ карточки, не только перевода (пробел G6).
9. **Privacy/consent (R5, пробел G9):** per-card Dicta-refine (Эпик 2) = исходящий текст → опт-ин, явная офлайн→онлайн граница, скрыт офлайн. OPFS-стор статуса (Эпик 4) — вне любого research/graph-экспорта (`privacy-smoke`).
10. **Норма деплоя:** бамп `sw.js CACHE_VERSION` + тост «Обновить» на любое shell-изменение; прод-верифи Node-fetch/браузер (не Windows-curl на не-ASCII).

**Опционально (пробел G5, измерение исходов):** в Зале НОЛЬ UX-инструментовки. Success-метрики Эпиков 4–6 = «фича есть», не «исход сдвинут». Privacy-preserving in-Room метрики удержания/сходимости-статуса = отдельный опц. трек (сначала PROPOSE, Direction-11 приватность).
