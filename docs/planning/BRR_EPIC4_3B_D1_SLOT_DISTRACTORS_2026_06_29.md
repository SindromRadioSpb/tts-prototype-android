# Эпик 4.3b · Phase D1 — полный slot-inflection дистракторов (R10-моат) · 2026-06-29

**Статус:** APPROVED (владелец 2026-06-29: «ядро-рантайм + семантика L4»). Родитель: `BRR_EPIC4_3B_RECALL_CLOZE_2026_06_28.md` §Phase D. Память [[project_brr_ux_audit]].

## Зачем
В MC сейчас правильный вариант — это склонённая форма из предложения, а дистракторы — формы кандидатов «как есть» в тексте → часто видно, что «грамматически подходит только ответ» (form-tell), и можно угадать по форме, не зная значения. D1 склоняет дистракторы в ТУ ЖЕ грамматическую позицию (слот), что и пропуск → MC проверяет ЗНАЧЕНИЕ. Моат R10: невозможно в generic-приложениях без ивритского морфо-движка.

## Measure-before-code (R10) — замеры 2026-06-29 (15 работ, 1278 MC; харнесс в scratchpad → формализуется в `reader-cloze-audit.js`)
- **Базовый охват «чистого» slot-inflect: ~24%** (слот ответа однозначно разрешим ∧ ≥3 слот-форм), form-tell в **~34%** вопросов.
- **Декомпозиция потерь разрешимости слота:** нет-парадигмы **0%** (словарь покрывает всё → ре-бейк/расширение НЕ нужны); нет-совпадения-ячейки **44.4%** (доминанта); синкретизм 18.2%; гомограф-`ambiguous` 15.4%; male/haser ~1.5%.
- **Решающее:** 61% MC-ответов несут проклитику; **снятие ОДНОЙ проклитики восстанавливает 90% «нет-совпадения»** → остаток всего **4.2% MC**. ⇒ потолок — рантайм-обработка строки, не данные.
- **Банк (L3):** наивный «выбрать-потом-проверить» занижает; правильно «фильтр по наличию слота → ранжирование» из словаря-в-памяти → producible≈100%.
- **Вывод:** рантайм-рычаги (проклитика + пин-по-лемме + синкр-accept + банк) → прогноз **~85–90%** чистых слот-вопросов; ре-бейк (L5) и новый shipped-артефакт НЕ нужны.

## Архитектура D1-v1 (рантайм · offline · R1 · R11 · детерминизм)
1. **Слот-резолвер (движок, чистый `findSlot(paradigm, niqqud)` → {slot, proclitic, count}):** пин парадигмы по СОБСТВЕННОЙ лемме изучаемого слова (`card.paradigm` от `resolveCore`/`pidMap`, игнор кросс-парадигма-`ambiguous`) → снять ведущую проклитику (ו/ה/ב/כ/ל/ש/מ) по детерминир. правилам, гард база≥2 (R10, без одно-букв. ложных снятий) → матч стема к `cells` (strip-niqqud + финал-норм) → слот. Синкретизм (count>1) → детерминир. выбор. Нет совпадения/слота → null (откат).
2. **Банк дистракторов (L3) = POS-индекс над `ensureEngine().pidMap`** (9279 парадигм уже в памяти; строится 1 раз, кэш). Кандидаты = same-POS леммы, у которых ЕСТЬ нужный слот (`cells[slot].he`).
3. **L4 семантическое ранжирование:** score = same-root-семья ≫ пересечение поля по RU-глоссам (общий значимый токен) > дет. тай-брейк. → дистракторы из близкого поля → MC на РАЗЛИЧЕНИЕ значения.
4. **Формы вариантов = ГОЛЫЕ слот-формы для ВСЕХ 4** (`cells[slot].he`): correct = форма ответа, дистракторы = формы кандидатов. **R1: только реальные ячейки Pealim, проклитику НЕ дорисовываем** (исключает ложную огласовку проклитики); все параллельны → нет proclitic/niqqud-tell. Дедуп по отображаемой форме, ни одна ≠ ответу.
5. **Откат к текущему B1** когда слот не разрешён / <3 годных форм → без регресса (R11 do-no-harm). Ответ — реальная форма (R11, не подмена). Детерминизм (lookup/ранжирование детерминированы, seed по индексу).

## Вшивание (на коде)
- `reader-morph.js`: pure `findSlot` (export, Node-тест) · async `buildMcSlotOptions(answerCard, n)` (использует внутр. pidMap-банк + findSlot + L4-ранж; возвращает `{correctHe, options:[he…]}` или null) (export).

Reveal согласован.
