# Концепция «Отмечать любое слово — честно» (статус-окраска для служебных + не-из-словаря) + back-nav семьи корня

**Дата:** 2026-06-27 · **Статус:** ✅ РЕАЛИЗОВАНО — T-a (v3.11.14) + T-b (v3.11.15) + back-nav (v3.11.13). См. «СТАТУС РЕАЛИЗАЦИИ» в конце.
**Родитель:** `BRR_EPIC4_RETENTION_LOOP_*` (петля удержания) · палитра `docs/research/word-status-palette/2026-06-27/`. Роли R1·R2·R3·R4·R5·R8·R9·R10·R11.
**Поверхность:** `reader-morph.js` + `library.html`-CSS (+ возможно `local-db.js` для manual-gloss). Room-only; resolver НЕ трогаем.

## Проблема (измерено 2026-06-27)
Селектор «Мой статус» и long-press-popover показываются на ЛЮБОМ слове (lemmaKey есть всегда: `על#particle`, `בנימה#`), статус сохраняется — но `decorateWords` красит ТОЛЬКО уверенные (label `exact|likely`). → на служебном/неизвестном слове можно выбрать цвет, но он не применится = **вводит в заблуждение**. При этом владелец прав: пользователь МОЖЕТ учить служебные слова и слова не из словаря — это естественно, и должен мочь (а) отметить их статусом-цветом, (б) для неизвестного — вписать хотя бы перевод вручную.

## Ключевой ролевой инсайт (R10 × R11)
**Честный гейт нужен для DERIVED-статуса** (SRS/Anki): не красить неуверенно-распознанное, т.к. можем мис-атрибутировать чужой лемме. **Но USER-SET статус — явное утверждение пользователя об ЭТОЙ форме**, он обходит неуверенность резолвера (пользователь — авторитет о своём знании). ⇒ **manual-статус красит по surface-форме НЕЗАВИСИМО от confidence; derived-статус сохраняет confident-гейт.** Нет риска мис-атрибуции, т.к. пользователь явно выбрал именно эту форму.

## Ролевой разбор
- **R1 лексикограф:** служебное `על` — реальная лексема; `נימה` — реальное слово, отсутствующее в НАШЕМ датасете (не не-слово). Оба легитимны для изучения. Не ФАБРИКОВАТЬ им корень/биньян/парадигму — хранить как surface-форму + пользовательский перевод.
- **R2 SLA:** учить служебные (высокочастотные!) и редкие контентные — естественно и ценно. Самооценка пользователя — авторитетный сигнал для чтения. Минимум для неизвестного — ручной перевод.
- **R3 граф:** ключ статуса: уверенные → по канонической лемме (как сейчас); неуверенные → **по surface-форме** (честная единица — не претендуем на лемму; зеркалит FTS-урок «гомографы не схлопываем → по форме»).
- **R4 premium-UX:** «выбрал цвет — не применился» = broken-promise, чинить. Путь: сделать так, чтобы применялся (красить по surface для manual). Для неизвестного — честный микро-лейбл «нет в словаре — отметь сам / добавь перевод» + фрикшенлесс-ввод. Провенанс пользовательского ≠ машинного.
- **R5 рынок:** LingQ красит ЛЮБОЕ помеченное слово (по форме), вкл. непарсящиеся. Это table-stakes; без этого мы отстаём.
- **R8 graded:** «стена сходится к чистому» работает только если ВСЕ слова трекаемы; иначе всегда остаётся неокрашенный шум.
- **R9 authority/provenance:** пользовательский статус/перевод помечать как user-provided (не выдавать за машинный). derived≠asserted≠user.
- **R11 do-no-harm:** **прецедент** — manual-surface-статус НЕ перебивает confident-lemma-статус там, где слово уверенно распознано (surface-fallback только для неуверенных). Иначе surface-статус мог бы «протечь» на confident-гомограф. Manual красит surface только когда резолвер неуверен.

## Премиум-концепция «Track any word — honestly»
1. **Manual-статус применяется к ЛЮБОМУ слову.** Уверенные → ключ-лемма (как сейчас). Неуверенные (function/unknown) → ключ-**surface**.
2. **decorateWords красит:** confident → по lemma-статусу (как сейчас); unconfident → по **surface-статусу ТОЛЬКО если пользователь явно его поставил** (derived остаётся не-крашеным). Прецедент R11: lemma-статус приоритетнее surface там, где слово уверенно.
3. **Нет misleading:** manual теперь всегда применяется → селектор не пустое обещание.
4. **Ручное обогащение неизвестного:** карточка неизвестного слова даёт «добавить перевод» (минимум) → пользовательский глосс на карточке, помечен «ваш» (R9). (Связь с word_study-заметкой — развилка C.)
5. **Провенанс:** user-marked статус/перевод визуально отличается от машинного.

## Пункт 2 — навигация «назад» в семье корня
Тап по чипу «Слова от этого корня» открывает карточку слова, заменяя текущую, БЕЗ «назад» (приходится закрывать и тапать заново; можно проваливаться многократно). **Концепция:** back-стек в карточке — каждый drill (чип→карточка) пушит предыдущую; «‹ Назад» в шапке карточки последовательно возвращает; пустой стек → стрелка скрыта. Самодостаточно, Room-only.

### Фаза T-a — «manual красит любое» (core, меньше)
1. **decorateWords:** снять жёсткий confident-гейт с ПОИСКА цвета. Правило:
   - **confident** слово → как сейчас: `STATE_CLASS[states[lemmaKey] || 'new']` (уверенные-непросмотренные = синий по умолчанию).
   - **unconfident** слово (function/unknown) → красить ТОЛЬКО если есть запись `states[surfaceKey]` (т.е. пользователь ЯВНО отметил статус ИЛИ сохранил заметку); **НЕ** дефолтить в 'new'. Без engagement → plain (честно).
   - `surfaceKey` = `NA.lemmaKey` неуверенной карточки (= `surface#pos`/`surface#`), тот же, что использует setWordStatus/getKnownWordStates → выравнено.
   - **R11-прецедент:** confident → lemma-key (surface-статус не течёт на confident-гомограф); surface-key только для unconfident.
2. **Гейт:** smoke — (a) unconfident+manual('known') → красится по surface; (b) unconfident без engagement → НЕ красится; (c) confident → по lemma-key (как сейчас). Расширить `reader-morph` + регрессия `על`/`בנימה` (без статуса — plain; со статусом — красятся).
