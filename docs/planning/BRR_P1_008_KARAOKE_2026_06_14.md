# BRR-P1-008 — Karaoke (sentence-level text↔audio sync) — RECON-FIRST DESIGN (на утверждение)

**Статус:** дизайн на утверждение (2026-06-14). Направление ② выбрано владельцем. Кода пока нет.
**Поверхность:** `public/library.html` Зал. **index.html НЕ трогать.** Роли: R4 (UX/RTL), R2 (восприятие), R5 (parity).

## Context — зачем
Единственная reading-фича, которой у нас нет, а у конкурентов (LingQ/Beelinguapp/StoryHebrew) есть: подсветка
читаемого предложения синхронно с аудио. Завершает премиум in-reading опыт после ① (леса).

## Recon (что УЖЕ есть — измеримый факт по коду)
- `attachRowAudio` (reader-core.js:390–521) играет ОДНУ строку (тап ▶ или тап ячейки) и **уже вешает
  `.row-playing` на `<tr>`** (:412), снимает на `ended` (:405, mode 'audio') и speech `onend` (:436).
- `reader-core.css` (грузится Залом) **уже стилизует** `.row-playing` (фон строки + левый акцент, :723/:735–736).
  ⇒ **подсветка одной играющей строки уже работает в Зале.**
- 3 тира аудио: tier-1 кэш `/api/audio/:assetKey` (канон — keyless/бесплатно), tier-2 BYOK GCP, tier-3 браузерный
  SpeechSynthesis (keyless fallback). `attachRowAudio` — **Room-only** (у index.html свой аудио-путь; parity-гейт
  держит только `buildBilingualTableHtml`, не attachRowAudio).
- **ПРОБЕЛ:** нет непрерывного потока (нет auto-advance к следующей строке), нет авто-скролла, нет кнопки «читать
  вслух». **Word-level тайминга НЕТ** (бейкнутые клипы без SSML-меток) → только **sentence-level**.

## Решение (sentence-level karaoke)
### D1 — Непрерывный режим в `attachRowAudio` (reader-core.js, Room-only, обратносовместимо)
- Добавить `playAll(startIdx)`, `stop()`, опц. `onRowChange(idx)` в возвращаемый handle.
- На завершение строки (и `ended` audio, и `onend` speech), если continuous и не остановлено → играть следующую
  **играбельную** строку (skip пустых). Чистая функция `nextPlayableIdx(rows, idx)` (Node-тест).
- Переиспользует существующий `play(idx)` + `.row-playing`. Тап одной строки (текущее поведение) — без изменений.
### D2 — Контрол в Зале (library-ui.js + library.html)
- Кнопка в reader-bar: **«▶ Читать вслух»** → старт continuous с верхней строки; переключается в **«■ Стоп»**.
  (Тап ▶ конкретной строки остаётся одиночным.)
### D3 — Авто-скролл (R4)
- На `onRowChange` — `scrollIntoView({block:'center', behavior:'smooth'})` играющей строки, чтобы караоке-строка
  оставалась видимой. Останавливаем авто-скролл при ручном скролле пользователя (флаг) — без борьбы со скроллом.
### D4 — Подсветка
- Переиспользуем `.row-playing` (уже есть). При необходимости чуть усилить контраст для караоке (отдельный класс
  `.row-karaoke` на mount при continuous, если стандартной подсветки мало — решим на скрине @380px).

## Scope OUT (честно)

корпус без аудио = tier-3 браузер-речь (keyless). Честно отметить в «О Зале»/подсказке.

## Риски/митигации
- Браузерная речь: голос he может отсутствовать на устройстве (tier-3 best-effort, как сейчас) — честный фолбэк.
- Авто-скролл vs ручной скролл пользователя → флаг «пользователь скроллит» приостанавливает авто-скролл.
- Длинный текст continuous через BYOK → стоимость; митигация: канон tier-1 бесплатно, корпус tier-3 keyless.
