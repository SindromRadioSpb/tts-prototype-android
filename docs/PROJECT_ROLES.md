# Роли проекта (R1–R11) — оптика для дизайн- и качество-решений

> **Назначение.** Для LinguistPro мы приняли пять экспертных ролей-линз. Любое
> содержательное решение по продукту/коду (новая фича, UX, морфология, граф,
> качество данных, спорный trade-off) прогоняется через релевантные роли, а
> кросс-режущие — через все пять с финальным синтезом. Цель — **бескомпромиссное
> премиальное качество, без заглушек**. Эти роли — не формальность: они ловят
> ошибки до пользователя и держат планку.
>
> **Как применять (автоматично):** при планировании/ревью называйте роли явно
> («R1: …», «R4: …»), как это делается в планах и предложениях. Для развилок —
> представляйте варианты с разбором по ролям и рекомендацией. Не нужно ждать
> напоминания пользователя — это рабочая норма проекта (см. CLAUDE.md).

---

## R1 — Ивритский лексикограф / морфолог
**Мандат:** корректность языковых данных иврита.
- Биньян / гизра / мишкаль; корень определяется по данным (Dicta morphId, hspell), **не угадывается** — омографы делят корень между частями речи (שבת сущ. vs לשבות глаг.).
- Формы **не выдумываются**: парадигмы — из авторитетного источника (Pealim); если нет — честное «нет таблицы / не изменяется», ссылка на перепроверку.
- Различение ktiv male/haser, огласовка, prefix-сегментация проклитик.
**Красные флаги:** выдуманная форма, «paal по умолчанию» для существительного, корень для служебного слова/имени собственного, неверная дизамбигуация омографа.

## R2 — Методист SLA (second-language acquisition)
**Мандат:** обучающая ценность и удержание.
- Для содержательных слов — парадигмы/корень-как-спина; для **служебных слов — употребление** (примеры из текстов, коллокации, позиция), а не формы.
- Прогрессивное раскрытие, не перегружать; данные питают cloze/квизы/SRS.
- Аудио + транслитерация с ударением как часть запоминания.
**Красные флаги:** информация ради информации; перегруз; «тупик» там, где нужен следующий шаг обучения; потеря живого контекста.

## R3 — Архитектор графа знаний
**Мандат:** честная структура связей.
- Узлы корней/биньянов точны (из декода); сущности (NER) — отдельные честные узлы (имя собственное ≠ корень).
- Рёбра по ID, не по тексту; приватность графа — гейт (`privacy-smoke`).
**Красные флаги:** ложный узел/ребро; служебное слово как «корень»; протечка приватных данных в граф.

## R4 — Премиум-UX (mobile-first, RTL)
**Мандат:** ощущение премиального продукта.
- **Скриншот @380px перед UI-коммитом** (см. CLAUDE.md workflow); RTL-иврит корректен.
- **Провенанс везде** (источник данных, «match» vs «подобрано»), честные пустые/офлайн-состояния, **никаких тупиков и «второго сорта»** — единые компоненты для всех видов слов.
- Прогрессивное раскрытие (аккордеоны), переиспользование эстетики (ячейка формы + ударение + тап-озвучка).
**Красные флаги:** «Таблица не найдена» как тупик; неясные подписи кнопок; перегруженная карточка; UI, не проверенный на 380px.

## R5 — Продукт / рынок
**Мандат:** конкурентная планка и стратегия.
- Эталон — Pealim/Reverso (таблицы, аудио, примеры); матчим **в приложении + офлайн**, внешние источники как «второе мнение».
- Offline-first — ядро ценности; согласие (consent) на любой исходящий текст.
**Красные флаги:** функция хуже бесплатных аналогов; онлайн-зависимость там, где ожидается офлайн; игнор ToS/приватности.

## Расширенные роли библиотеки (BRR)

> R6–R8 введены при пивоте в «Издательство + Читальный зал» (полные определения —
> `docs/strategy/BEN_YEHUDA_LIBRARY_READING_ROOM_STRATEGY.md`, §«Новые роли»);
> здесь — навигационная сводка. R9 определён ниже полностью.

- **R6 — Куратор-библиотекарь / collection-developer:** библиотека как система (комплектование,
  схема метаданных, левелинг, discoverability, дедуп, **атрибуция per work**). 🚩 дамп вместо библиотеки.
- **R7 — Литературовед-гебраист / редактор:** суждение о каноне, **честность с регистром/эпохой**,
  редакторские интро, порядок чтения. 🚩 тяжёлую архаику подавать «для начинающих».
- **R8 — Дизайнер graded-reading:** проходимость, скаффолдинг **с угасанием**, on-ramp, «что дальше».
  🚩 скаффолдинг, который не убирается; неградуированная свалка.

## R9 — Инженер authority-control / связанных данных (LOD)
**Мандат:** честное, воспроизводимое обогащение метаданных из внешних авторитетных источников
(Wikidata/VIAF) — введён 2026-06-10 для coverage-модели на 26K (BRR-P1-014 A2; эпоха из дат жизни).
- **Идентичность по стабильному ID:** сущность резолвится к внешнему якорю (Wikidata QID есть у 90.2% авторов);
  варианты написания имени мёржатся **по QID, не по строке** (анти-коллизия: разные люди с одним именем = разные QID).
- **Derived ≠ asserted ≠ curated:** производные поля (эпоха из P569/P570, язык, пол) выводятся по
  **документированным правилам**, помечаются `provenance:derived` (источник QID + правило + дата запроса) и
  **никогда** не выдаются за asserted (CSV) или curated (человек). Нет данных → честное `unknown`, не угадка (с R1).
- **Эпистемика производной эпохи:** литературный период ≈ **флоруит** (≈ год рождения + ~35; fallback год смерти − ~30),
  не «год смерти» (иначе Бялик р.1873/у.1934 уехал бы в «подмандатный»); пограничные случаи помечаются low-confidence.
  Границы бакетов — зона R7; механика вывода и провенанс — R9.
- **Вежливость + воспроизводимость:** батч (SPARQL `VALUES`), описательный User-Agent, backoff, **on-disk кэш** →
  re-run не долбит публичный сервис; обогащение детерминированно и пересобираемо офлайн из кэша/коммитнутого артефакта.
**Красные флаги:** слияние разных людей по совпадению имени; производная дата/эпоха, выданная за известную;
хаммеринг публичного API без кэша/бэкоффа; необратимое/невоспроизводимое обогащение; «придуманные» даты.

## R10 — Вычислительный морфолог (дизамбигуация и оценка)
**Мандат:** честная разрешимость ивритской морфологии в тап-резолвере Зала — выбор ВЕРНОГО чтения
гомографа/проклитики/суффигированной формы и **измеримая** оценка качества. Введён 2026-06-11 (BRR-P1-009,
после measured-QA: на work 34704 17% слов = «карточка даёт глосс, ссылка ведёт на поиск», доминирующий
класс — служебные слова с ЧУЖИМ гомограф-глоссом: אֵין→«уничтожить», עָלֵינוּ→«лист»).
- **Изоляция ≠ контекст:** form-first над офлайн-Pealim разрешает форму в вакууме; на гомографах это
  даёт уверенно-выглядящий ЛОЖНЫЙ смысл. Контекстная истина (Dicta/частотность) — потолок, но её
  отсутствие НЕ оправдывает выдачу чужого глосса за ответ.
- **Честный гейт глосса (make-or-break):** когда чтение получено только meaning-fallback (нет
  form-match с pealim_id) И слово закрытого класса (стоп-лист ИЛИ детерминированная сегментация
  предлог/местоимение+прон.-суффикс, рефлексив עצם, числительные) — НЕ утверждать одиночный глосс;
  показать честное служебное чтение или «не определено офлайн → уточните по ссылке». Лучше «не знаю»,
  чем уверенная ошибка (с R1).
- **Сегментация без угадывания:** проклитика (ו/ש/ה/כ/ב/ל/מ) и прон.-суффиксы (־נו/־הם/־יה/…) снимаются
  только по детерминированным правилам с базой длины ≥2 (одно-буквенные ב/כ/ל/מ-базы — запрещены:
  לִבֵּנוּ=«сердце», не предлог). Каждое снятие — фолс-позитив-бюджет, измеряется.
- **Клитический граничный анализ + политика воздержания (Phase-2 проклитик; включает свёрнутую сюда роль «морфо-сегментатор», решение владельца 2026-07-01):** R10 владеет ГРАНИЦЕЙ слова — упорядоченная FSA-грамматика приставок (`ו?`→`{ש|כש|לכש|מש}?`→`ה?`→один из {ב,ל,כ,מ}; ו только внешний, מ не сливает артикль), агглютинативный стэкинг, whole-word do-no-harm guard (ktiv-robust skeleton), огласовка ТОЛЬКО как уверенность (не существование), **воздержание при ничьей/глубине≥3/отсутствии огласовки**, раздельные метрики existence-precision (≥99) / labeled-seg (≥95) / recall (негейтящий) против **НЕЗАВИСИМОГО** gold (не Dicta-vs-Dicta), эскалация офлайн→bake-time/Tier-3. **R11 владеет never-segment tripwire** (хард-негатив בית/משה/ביחוד → ноль приставок); **R1 — смысл/фоссилии/живость клитики**; **R4 — вето на surfacing уверенности.** (Энклитики вне scope.) Канон: `BRR_PROCLITIC_PHASE2_RECON_2026_07_01.md`.
- **Измеримость обязательна:** любой рычаг (стоп-лист / сегментация / перебейк никуда / Dicta-режим)
  оценивается на реальных работах против silver-стандарта (Dicta-контекст локально) с раздельными
  recall/false-positive; «улучшение» без замера на корпусе — не принимается. Silver ≠ gold (Dicta тоже
  ошибается на архаике) — помечать доверие.
- **Деградация — честная:** если контекст-провайдер вернул пусто на непустой иврит, это `degraded:true`,
  а не молчаливый успех; клиент откатывается на офлайн-гейт осознанно. (Урок: «прод-egress к Dicta закрыт»
  оказался артефактом Windows-curl, калечившего UTF-8 → измеряй корректным инструментом, Node/браузер.)
**Красные флаги:** уверенный глосс чужого гомографа на служебном слове; сегментация-угадайка (одно-букв. база,
ложное снятие суффикса); «стало лучше» без замера recall/FP на корпусе; silver, выданный за gold;
молчаливый пустой ответ вместо честной деградации.

## R11 — Регрессолог-текстолог (do-no-harm / источник-истины)
**Мандат:** любое «улучшение» резолвера/обогащения (Tier-3 контекст, газеттир, демоция, перебейк
никуда, будущие режимы) должно быть **нетто-монотонным** — добавлять информацию, но **никогда не
превращать ВЕРНОЕ обоснованное чтение в неверное**. Введён 2026-06-26 после measured-бага: Tier-3
контекст-режим молча переписывал корпусное בֹּקֶר «утро» → בָּקָר «крупный рогатый скот» (Dicta берёт
частотный гомограф на голой архаике), противореча колонке огласовок, что видит читатель. Прошлые
линзы (R1/R10/R9) аудируют корректность ОДНОГО чтения; ни одна не ловила деградацию уже-верного,
кросс-поверхностную рассогласованность и зависимость замера от проверяемого источника.
- **Источник-истины (precedence):** корпусный никуд (что читатель ВИДИТ; запечён с контекстом всей
  работы) старше живой односложной Dicta-переогласовки. Улучшение **дополняет**, не перезаписывает
  обоснованное чтение. Решающее офлайн-чтение (form-first match корпусного никуда) не отменяется
  внешней догадкой; в крайнем случае — показать как альтернативу, не подменять.
- **Кросс-поверхностная согласованность:** тап-карточка НЕ должна противоречить колонке огласовок /
  переводу, на которые читатель уже смотрит. Рассогласованность поверхностей — сама по себе P0
  (эрозия доверия к моату), даже если каждая поверхность «по-своему правдоподобна».
- **Независимость оракула:** нельзя мерить безопасность фичи тем же источником, которому фича доверяет.
  Если Tier-3 доверяет Dicta, Dicta **не может** быть регресс-оракулом (слеп к собственным ошибкам —
  «согласен сам с собой»). Нужен независимый эталон: корпусный никуд / человеческий gold.
- **Адверсарный head-регресс:** мерить ВРЕД на уже-верном «head», а не только пользу на «tail». Искать
  худший гомограф, который новый режим ломает (частотный-неверный поверх редкого-верного; архаичный
  регистр). «Улучшил tail» без замера регресса на head — не принимается (с R10).
**Красные флаги:** режим, перезаписывающий решающее корпус-обоснованное чтение; карточка ≠ колонка;
валидация фичи тем же источником, которому она доверяет; «0 регрессий» от оракула, слепого к своим
ошибкам; замер только пользы на tail без замера вреда на head; необратимая подмена огласовки, что видит читатель.

## Протокол синтеза
1. **Точечное решение** → релевантная роль (или 2).
2. **Кросс-режущее** (фича/редизайн) → все пять, затем синтез: выбрать вариант, привить лучшие идеи остальных, явно назвать компромиссы.
3. **Развилка с участием пользователя** → представить варианты с разбором по ролям + рекомендация (AskUserQuestion), решение — за пользователем.
4. **Инвариант:** «бескомпромиссное качество, без заглушек» (директива владельца). Если роль поднимает красный флаг — не замалчивать, чинить или честно коммуницировать ограничение.
