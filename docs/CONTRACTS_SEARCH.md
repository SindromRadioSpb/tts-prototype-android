# CONTRACTS — Rows Search PRO

## 1) Цель
Сделать поиск по строкам/предложениям качественным и предсказуемым:
- Нормализация иврита (критически важно)
- Авто-детект языка запроса
- Snippet и Highlight обязательны
- Одинаковая логика перехода (jump) из Rows и Notes

## 2) Термины
- **raw_text** — исходный текст (как хранится/показывается)
- **norm_text** — нормализованный текст (для поиска/индексации)
- **query_norm** — нормализованный запрос пользователя
- **match** — совпадение в norm, отображается как highlight на raw

## 3) Языковые режимы поиска
Режимы:
- `hebrew`
- `russian`
- `english`
- `auto`

### 3.1. Авто-детект языка запроса (детерминированно)
Алгоритм:
1) Посчитать долю символов:
   - Hebrew: диапазон `\u0590-\u05FF`
   - Cyrillic: `\u0400-\u04FF`
   - Latin: `A-Za-z`
2) Если Hebrew >= 0.25 → `hebrew`
3) Иначе если Cyrillic >= 0.25 → `russian`
4) Иначе если Latin >= 0.25 → `english`
5) Иначе → `auto` fallback к “plain contains” (или last-used).

## 4) Нормализация иврита (обязательная спецификация)
### 4.1. Цель нормализации
Сделать совпадение устойчивым к:
- никууду и огласовкам,
- пунктуации/кавычкам,
- конечным формам букв,
- лишним пробелам/разделителям.

### 4.2. Hebrew Normalization Rules (v1)
Вход: `raw_hebrew`  
Выход: `hebrew_norm`

Правила:
1) Удалить все никууд/диакритики:
   - диапазоны combining marks, в т.ч. U+0591..U+05BD, U+05BF, U+05C1..U+05C2, U+05C4..U+05C5, U+05C7
2) Привести конечные формы к базовым:
   - ך → כ
   - ם → מ
   - ן → נ
   - ף → פ
   - ץ → צ
3) Привести разделители к пробелу:
   - maqaf `־`, дефис `-`, слэш `/`, точки/запятые и т.п. → пробел
4) Удалить кавычки/гереш/гершайим как значимые символы:
   - `'`, `"`, `׳`, `״` → удалить или заменить на пробел (выберите единообразно: v1 = пробел)
5) Сжать пробелы:
   - несколько пробелов → один пробел
   - trim по краям
6) Привести к lower-case там, где применимо (для иврита не влияет, но для mixed строк — да).

### 4.3. Где хранится hebrew_norm
Минимальный контракт:
- Для каждой searchable сущности (row/sentence) должна быть доступна нормализованная строка:
  - либо колонка `hebrew_norm`,
  - либо таблица `search_index` (см. DB_SCHEMA.md).
- Должен существовать индекс (или FTS), иначе поиск деградирует.

## 5) Нормализация русского/английского (минимум)
Для ru/en:
- lower-case
- trim
- заменить пунктуацию на пробел
- сжать пробелы

## 6) Совпадение (matching)
### 6.1. Matching contract
1) matching выполняется по `norm_text` против `query_norm`.
2) Возвращаемые результаты должны содержать:
   - `entity_id`
   - `raw_text`
   - `snippet` (обязателен)
   - `highlights` (обязателен)

### 6.2. Snippet contract
- Длина: 60–160 символов (конфиг)
- Snippet должен быть “окно вокруг совпадения”
- Для RTL: окно формируется по raw, но границы рассчитываются по match position в norm
- Если несколько совпадений — выбрать наиболее релевантное (первое или самое плотное)

### 6.3. Highlight contract
- Отдавать список диапазонов или список фрагментов:
  - вариант A: `[ { start, end } ]` на raw_text
  - вариант B: raw_text с разметкой `<mark>` (не рекомендуется для хранения)
Рекомендуется вариант A.

## 7) Интеграция с навигацией
Любой результат поиска должен содержать:
- `target` (см. CONTRACTS_NAVIGATION.md)
- `context` (включая q/lang/filters)

Это обеспечивает “Back to results” и одинаковый jump-to-sentence.

- Морфологический поиск (лемматизация иврита) — v2+
- Синонимы/раскладка клавиатуры — v2+
- Фонетический поиск — v2+
