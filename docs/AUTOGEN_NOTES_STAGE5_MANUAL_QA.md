# Stage 5 (Concept B — one-tap «читай-и-собирай») — ручной смоук-чек

**Версия:** SW `v3.8.7-collect` (commit `d14d557`). **ПРОД:** `https://linguistpro.kolosei.com`.
**Девайс:** mobile-first — тестировать на телефоне или в DevTools @380px, RTL. Десктоп — вторично.

> Перед началом на ПРОДе: `Ctrl/Cmd-Shift-R` (hard reload), чтобы новый Service Worker
> `v3.8.7-collect` активировался. Проверка версии: DevTools → Application → Service Workers,
> либо в консоли `caches.keys()` → должен быть `…-v3.8.7-collect`.

---

## 0. Карта UI (где что лежит)

- **Тумблер сбора** — чекбокс **«Собирать слова одним тапом (при чтении)»**, сразу под селектом
  «Авто-строить знания из текстов», в **едином** модале настроек морфологии **«🔤 Морфология: словарь»**.
  Открывается кнопкой 🔤 в шапке главного экрана (а также из справки по заметкам → «Статус словаря…» —
  с v3.8.9 ведёт в тот же модал; легаси-дубль «Морфология иврита» удалён).
- **Сбор слова** — открыть текст → тап по кнопке 📝 (заметки) в нужной строке → панель «📝 Заметки к
  строке» → раздел **«+ Новая заметка»** → кнопка **«🔤 Слово отсюда»** → выбор слова → **тап по слову**.
- **Обогащение текста** (предусловие для офлайн-сбора) — карточка текста в библиотеке → кнопка
  **«Обогатить (Dicta)»** (`data-act="enrich"`). Без обогащения у строки нет морфологии.

---

## 1. Предусловия

- [ ] Открыт текст на иврите, в библиотеке (LOCAL_MODE / OPFS).
- [ ] Текст **обогащён морфологией** (Dicta) — иначе сбор уйдёт в фолбэк (см. Кейс 6).
- [ ] Тумблер «Собирать слова одним тапом» по умолчанию **ВЫКЛ** (чистая первая загрузка).

---

## 2. Кейсы

### Кейс 1 — тумблер по умолчанию ВЫКЛ → старое поведение (редактор)
1. Не включая тумблер, строка → 📝 → «🔤 Слово отсюда» → тап по слову.
- [ ] **Ожидаемо:** открывается обычный **редактор** заметки (как раньше), морфология предзаполнена.
      Никакой заметки молча не создаётся.

### Кейс 2 — включение тумблера
1. Открыть «🔤 Морфология иврита», поставить галочку «Собирать слова одним тапом».
- [ ] Галочка ставится, подпись и подсказка читаются, вёрстка @380px не разъезжается (RTL ок).
2. Закрыть модал, перезагрузить страницу, снова открыть модал.
- [ ] **Ожидаемо:** галочка **сохранилась** (persist в localStorage `notesCollectMode_v1`).

### Кейс 3 — happy path: один тап = заметка + тост с «Отменить»
1. Тумблер ВКЛ. Строка → 📝 → «🔤 Слово отсюда» → тап по **знаменательному** слову (глагол/сущ.).
- [ ] **Редактор НЕ открывается.**
- [ ] Внизу тост **«✨ Собрано: <слово> · <перевод>»** с кнопкой **«Отменить»**.
- [ ] Счётчик заметок 📝 у строки **вырос** (без перезагрузки).
2. Открыть строку → 📝.
- [ ] Заметка появилась в разделе **«✍ Ваши»** (НЕ в «✨ Авто») — т.к. source=`'curated'`.
- [ ] У заметки заполнены поля: слово, огласовка, корень/лемма, часть речи, перевод (если есть в словаре).

### Кейс 4 — undo только что созданной заметки
1. Сразу после Кейса 3 (пока тост виден) нажать **«Отменить»**.
- [ ] Тост «Отменено».
- [ ] Заметка **исчезла** (счётчик 📝 вернулся к прежнему; в «✍ Ваши» её нет).

### Кейс 5 — повторный сбор того же слова → undo снимает только это вхождение
1. Тумблер ВКЛ. Собрать слово W (Кейс 3) — НЕ отменять.
2. В **другой строке** того же текста собрать **то же** слово W (та же лемма/смысл).
- [ ] Новой отдельной заметки **не появляется** (дедуп по канонической лемме) — добавляется вхождение
      к существующей; тост снова показан.
3. Нажать «Отменить» у второго тоста.
- [ ] **Ожидаемо:** заметка **остаётся** (она была раньше), но **снимается именно второе вхождение**
      (счётчик/occurrences возвращаются к состоянию после шага 1, а не к нулю).

### Кейс 6 — фолбэк без морфологии (R1: ничего не выдумываем)
1. Взять текст/строку **без** обогащения (или новый необогащённый текст). Тумблер ВКЛ.
2. Строка → 📝 → «🔤 Слово отсюда» → тап.
- [ ] Тост-предупреждение **«Морфология не готова — заметка через редактор»**.
- [ ] Открывается **редактор** (как в Кейсе 1). Молча «тонкая» заметка **НЕ** создаётся.

### Кейс 7 — функциональное слово / предлог
1. Тумблер ВКЛ. Собрать предлог/частицу/местоимение.
- [ ] Заметка создаётся; корень **пустой** (R1 — у служебных нет триконсонантного корня),
      часть речи определена (предлог/местоимение/…), без выдуманного глагольного корня.

### Кейс 8 — выключение тумблера возвращает редактор
1. Снять галочку. Собрать слово (Кейс 3-шаги).
- [ ] **Ожидаемо:** снова открывается редактор; сбор одним тапом отключён.

## 3. Регрессии (не должно сломаться)

- [ ] Обычное создание заметки через редактор (тумблер ВЫКЛ) работает как прежде.
- [ ] Существующие тосты (копирование ссылки, ошибки и т.д.) — без лишних кнопок и без изменений TTL.
- [ ] Разделы «✍ Ваши» / «✨ Авто» в панели строки рендерятся корректно; собранные заметки идут в «✍ Ваши».
- [ ] Поиск по заметкам находит собранную word-заметку (niqqud-insensitive).
- [ ] В Карте знаний собранное слово появляется узлом (после обновления данных карты).
- [ ] Нет ошибок в консоли (DevTools Console) на всём прогоне.

---

## 4. Сброс между прогонами

- Удалить собранные тестовые заметки: открыть строку → 📝 → у заметки «удалить»; либо отменять тостом
  сразу после сбора.
- Полный сброс окружения (НЕ на реальной библиотеке!): DevTools → Application → Storage → удалить OPFS,
  либо в чистом приватном окне.
