# Полный ручной тест-алгоритм: интеграция с Anki (A экспорт · B чтение-обратно · G1–G4 гарантии)

> Версия сборки под тест: **`v3.10.79-anki-g2-rich-export`** (HEAD `201415e`). Прогон выполняет владелец (не разработчик) на
> реальном Anki Desktop + телефоне. Каждый шаг: ЧТО нажать → ЧТО увидеть (наблюдаемо). Точные строки UI — в кавычках, искать
> дословно. Алгоритм выверен против фактического кода (grounding-воркфлоу 2026-06-19, адверсариальная сверка — verdict ok).

---

## 0) Предусловия / Setup

**0.1 — Обновить сборку ДО всего остального (иначе тест ложный).**
1. Открой приложение (`https://linguistpro.kolosei.com` или `http://localhost:3000`).
2. Если всплыл тост «Обновить» — нажми его. Не «отклоняй» и не делай мягкий reload: это оставит старый код.
3. DevTools (F12) → **Application → Service Workers**. Активный SW должен содержать **`v3.10.79-anki-g2-rich-export`**.
   Если нет — **Unregister**, затем полный hard-reload (Ctrl+Shift+R) и повтори.
4. **Проба свежести кода** (обязательно). DevTools → Console:
   - `window.AnkiSrsExport.MODEL_NAME` → должно вернуть **`LinguistPro Word v2`**.
   - `window.AnkiIdentity.LEMMA_TAG_PREFIX` → должно вернуть **`lp_lemma_`**.
   - Если `window.AnkiIdentity === undefined` → это ДО-hardening сборка; G1 тихо не сработает. СТОП, обнови
     (shell-файлы `db/anki-*.js`, `sql-wasm.js`, `jszip.min.js` грузятся БЕЗ `?v`, поэтому unregister+reload иногда не
     помогает — повтори, пока проба не пройдёт).

**0.2 — Anki Desktop + AnkiConnect (нужно для B, G1, G3, G4 — НЕ для A/мобайла).**
1. Установи **Anki Desktop**.
2. Add-on **AnkiConnect**: Anki → Tools → Add-ons → Get Add-ons → код **`2055492159`** → перезапусти Anki.
3. Allowlist: Anki → Tools → Add-ons → **AnkiConnect** → **Config** → в массив `webCorsOriginList` добавь свой origin —
   **`https://linguistpro.kolosei.com`** и/или **`http://localhost:3000`** (тот, с которого реально открыта вкладка).
   Сохрани → **перезапусти Anki**.
4. AnkiConnect слушает **`http://127.0.0.1:8765`** (`version:6`). Браузер ходит туда напрямую. Anki должен быть ЗАПУЩЕН
   во время B/G1/G3/G4.

**0.3 — Телефон** (A3 / B4): AnkiDroid (Android) или AnkiMobile (iOS). Для словесной TTS-озвучки нужен установленный
**he-IL** TTS-голос (Android — he-IL TTS-движок; iOS — ивритский Siri-голос).

**0.4 — Реальный профиль (критично — пустой профиль даёт ложный pass).**
Нужен профиль с **несколькими ②-словарными заметками (word_study)**, причём минимум у 2–3 слов:
- есть разрешимая парадигма (есть `pealim_id`/корень) → поле Conjugation непустое;
- первое вхождение слова связано с предложением, у которого есть **озвучка** (default-audio) → Example + `[sound:lp_<key>.mp3]`
  непустые.
Также нужен хотя бы один **текст с предложениями, у которых есть аудио** (для sentences-экспорта). Текст(ы) должны быть
**сохранены в Library** (иначе модальный экспорт остановится с тостом **«Сначала сохраните текст в Library»**).

## A) Экспорт .apkg

> Обе поверхности собираются ЦЕЛИКОМ в браузере (`window.AnkiApkg` = sql.js + jszip), данные не покидают устройство. Обе
> используют общие модели `db/anki-models.js`, поэтому карты сливаются при повторном импорте по одному GUID на лемму.

### A1 — Trainer GLOBAL (весь словарь, RICH)

1. Открой текст с несколькими ②-словарными заметками.
2. Открой **🎯 SRS Тренажёр**. Секция с заголовком **«Экспорт словаря в Anki»** и подсказкой **«Скачивает все словарные
   заметки одним .apkg (собирается в браузере — данные не покидают устройство). Повторный импорт обновляет карточки.»**
3. Нажми **«📦 Скачать .apkg (словарь)»**.

**Ожидаемо (наблюдаемо):**
- Кнопка → **«⏳ Собираю…»** → прогресс **«⏳ <готово>/<всего>…»** (живой счётчик: резолв парадигм + аудио).
- По завершении — тост **«Экспортировано <N> карточек · аудио <M> → LinguistPro-Words-<ГГГГ-ММ-ДД>.apkg»** (часть
  «· аудио <M>» — только если аудио было). Кнопка возвращается в **«📦 Скачать .apkg (словарь)»**.
- Скачался **`LinguistPro-Words-<сегодняшняя дата>.apkg`** (инфикс **`-Words-`**).

**Импорт в Anki Desktop:** File → Import → файл. Browse → колода:
- Колода **`LinguistPro::Words`**.
- Модель **`LinguistPro Word v2`**, ровно **11 полей** в порядке: **Word, Niqqud, Translit, Russian, Root, Binyan, POS,
  Conjugation, Example, Mnemonic, Audio**.
- Карта слова с известным спряжением показывает **таблицу спряжения** (Conjugation) + **пример-предложение** (Example);
  при «озвучке» — играет встроенное аудио, в медиа есть **`lp_<assetKey>.mp3`** (`[sound:lp_<assetKey>.mp3]`).
- Кол-во карт ≈ числу **уникальных лемм** (дедуп по лемме). Слово, у которого нет связанного предложения-вхождения, всё
  равно даёт карту — но с **пустым полем Example** и лицевой через **`{{tts he_IL:Word}}`** (внутренне — LEFT JOIN, так что
  автоген-заметка без вхождения тоже экспортируется; для тестера наблюдаемо именно «карта есть, Example пуст»).
- Слово БЕЗ аудио → лицевая через device-TTS **`{{tts he_IL:Word}}`**.

**Граничные тосты:**
- Нет словарных заметок → **«Нет словарных заметок (word_study) для экспорта.»**
- Слов >3000 → heads-up **«Большой словарь (<N> слов) — сборка с разборами и аудио может занять до минуты.»** (лимита нет).
- Модуль не загружен → **«Экспорт .apkg недоступен (модуль не загружен).»**
- Локальная база недоступна → **«Локальная база недоступна.»**
- Ошибка сборки → **«Не удалось собрать .apkg: <текст ошибки>»**.

### A2 — Модальный экспорт (per-text: words / sentences / both)

1. Открой **сохранённый в Library** текст → кнопка **«Anki»**. Модал с заголовком **«Экспорт в Anki (AnkiConnect)»**.
2. Подсказка модала: **«„📦 Скачать .apkg" работает везде (Desktop / Android / iPhone) — скачайте файл и импортируйте в
   Anki. „Экспортировать" отправляет напрямую в запущенный Anki Desktop (нужен AnkiConnect; Dashboard локально на
   127.0.0.1:3000).»**
3. Контролы:
   - **«Колода»** (placeholder `LinguistPro::A1`) — кастомное имя колоды.
   - **«Что экспортировать»**: **«Слова (②-база знаний)»** / **«Предложения»** / **«Оба типа»**.
   - **«Иврит на лицевой»**: **«Иврит (без огласовок)»** / **«Иврит+Никуд (с огласовками)»**.
   - ☑ **«Подсказка (тема/уровень)»** (default вкл.)
   - ☑ **«Не трогать карточки из других колод (Fresh-only)»** (default вкл.)
   - ☑ **«Озвучивать слова (GCP/устройство)»** (default вкл.)
4. Кнопка **«📦 Скачать .apkg»** (это `.apkg`, НЕ AnkiConnect-кнопка «Экспортировать»).

**A2a — «Слова».** «Что экспортировать» = **«Слова (②-база знаний)»** → **«📦 Скачать .apkg»**.
- Статус: **«Собираю .apkg…»** → **«Собираю .apkg (<всего> карточек · <M> аудио)…»** → **«Готово: <filename>»**.
- Тост: **«Скачано <total> карточек (слов <wc> · предложений <sc> · аудио <M>) → LinguistPro-<ГГГГ-ММ-ДД>.apkg»**.
  ВАЖНО: имя файла **`LinguistPro-<дата>.apkg`** — БЕЗ инфикса `-Words`.
- В Anki: колода **`LinguistPro::Words`**, модель **`LinguistPro Word v2`** (11 полей). Кастомное имя «Колода» применяется
  (одно-типный override).

**A2b — «Предложения».** **«Предложения»** → **«📦 Скачать .apkg»**.
- В Anki: колода **`LinguistPro::SRS`**, модель **`LinguistPro SRS Card v1`**, ровно **6 полей**: **Hebrew, Niqqud,
  Translit, Russian, Note, Audio**. Кастомное имя «Колода» применяется.

**A2c — «Оба типа».** **«Оба типа»** → введи кастомное имя в «Колода» → **«📦 Скачать .apkg»**.
- Один файл создаёт **ДВЕ** колоды: **`LinguistPro::Words`** (Word v2) + **`LinguistPro::SRS`** (SRS Card v1).
- В тосте счётчики разбиты: **«слов <wc> · предложений <sc> · аудио <M>»**.
- **Инвариант:** при «Оба типа» кастомное имя «Колода» **ИГНОРИРУЕТСЯ** — колоды принудительно `LinguistPro::Words` /
  `LinguistPro::SRS`. By design.

**Гейты модала:**
- Текст НЕ сохранён в Library → **«Сначала сохраните текст в Library»**, экспорт не идёт.
- Модуль не загружен → **«Экспорт .apkg недоступен (модуль не загружен).»** (тот же общий гейт, что и у глобального — не
  только global).
- Карт нет → **«Нет карточек для экспорта в этом тексте.»**
- Ошибка → тост **«Не удалось собрать .apkg: <msg>»** + статус **«Ошибка сборки .apkg»**.

**Замечание (не баг):** дропдаун «Иврит на лицевой» (plain/niqqud) `.apkg`-путь НЕ читает — у «Word v2» лицевая жёстко
`{{Word}}` с `{{Niqqud}}` ниже.

### A3 — Импорт .apkg на телефоне

1. Перенеси любой из скачанных `.apkg` (A1/A2) на телефон → открой в **AnkiDroid/AnkiMobile** → импорт.
2. **Ожидаемо:** те же колоды/модели (`LinguistPro::Words` / `LinguistPro::SRS`, 11/6 полей); таблица спряжения + пример
   рендерятся; встроенное **`[sound:lp_<key>.mp3]` играет всегда**; лицевая `{{tts he_IL:Word}}` звучит **только** при
   установленном he-IL TTS-голосе (иначе тишина — НЕ дефект экспорта). AnkiConnect на телефоне нет → push/Sync недоступны;
   `.apkg` — единственный путь, и он работает.

### A4 — Идемпотентность повторного импорта

1. Импортируй `.apkg` (например A1 global) → запиши число карт → импортируй **тот же** файл ПОВТОРНО.
2. **Ожидаемо:** Anki говорит **«updated»**, не «added»; число карт **не растёт** (стабильный GUID `word:<lemmaKey>`;
   омографы по лемме схлопываются); расписание существующих карт сохраняется.
3. **Ловушка:** если перед повтором ты изменил ключ леммы (POS / никуд / появился `pealim_id`) — GUID меняется
   (`pid:<id>` иначе `w:<word>#<pos>`), Anki создаст НОВУЮ карту. Это НЕ баг идемпотентности — сверь драйвер GUID.

## B) Чтение-обратно (read-back) — **Только Desktop, LOCAL_MODE**

> LOCAL_MODE включён по умолчанию у всех реальных пользователей; серверные Anki-роуты — 410 Gone. Всё идёт
> браузер→AnkiConnect напрямую (`POST http://127.0.0.1:8765`, `version:6`). На телефоне B недоступно целиком.

### B1 — Health + Push (AnkiConnect)

1. Запусти Anki Desktop (allowlist настроен). В приложении: сохранённый текст → **«Anki»**.
2. При открытии модала статус: **«Проверьте AnkiConnect и нажмите „Экспортировать".»** Health может стартовать сам.
3. **«Проверить AnkiConnect»**:
   - **«Проверяю AnkiConnect…»** → успех **«AnkiConnect доступен (version: <N>).»**
   - Провал: **«AnkiConnect недоступен: <msg>»** + CORS-подсказка **«— добавьте „<origin>" в webCorsOriginList AnkiConnect
     (Anki → Tools → Add-ons → AnkiConnect → Config) и перезапустите Anki.»**
4. **«Экспортировать»** (AnkiConnect push, НЕ `.apkg`):
   - Статус **«Экспортирую в Anki… (создаю/обновляю карточки)»**.
   - Успех: статус с разбивкой **«Готово (deck „<deck>", модель „<model>"): создано <N>, обновлено <N>, … с аудио <N>,
     перенесено в этот deck <N> card(s), дубликатов Note <N>, не найдено в Anki <N>, пустых пропущено <N> …»** + тост
     **«Экспорт в Anki выполнен»**.
   - Провал: **«Ошибка экспорта: <msg>»** (+ CORS-подсказка) + тост **«Ошибка экспорта в Anki»**.
5. В Anki Browse: колода создана, модель обеспечена, карты добавлены. У словесной карты теги: **`linguistpro`, `lp_word`,
   `lp_pos_<POS>`, `lp_lemma_<12hex>`, `lp_note_<localNoteId>`** (push ставит И `lp_note_`, И `lp_lemma_`).
6. **Fresh-only:** при ☑ существующие карты с тем же ивритом в ДРУГИХ колодах не двигаются; целевая получает одну карту на
   строку. «push ничего не переместил из старой колоды» — это и есть Fresh-only, не баг.

### B2 — Учёба в Anki + Синхронизация обратно

1. В Anki **позанимайся** картами (Good/Again), чтобы они вышли из `new` (reps>0).
2. В приложении (Anki запущен, origin allowlisted): текст → **«Anki»** → **«Синхронизировать из Anki»**.
   - Tooltip: **«Подтянуть результаты повторений из Anki: слова, которые вы учили в Anki, станут „известными"/„учу" здесь
     (питает граф знаний и i+1). Повторение остаётся в Anki.»**
   - Статус **«Синхронизирую из Anki… (читаю результаты повторений)»**.
   - Успех: **«Синхронизировано: <totalNotes> слов из Anki — <detail> · удержание <X>% (<R> повторов).»** где `<detail>` =
     **«обновлено <N>, новых выучено <N>, отложено <N>, оставлено своё <N>, ещё не учили <N>, нет в библиотеке <N>»** (или
     **«изменений нет»**). Тост **«Синхронизировано из Anki: <detail>»**.
3. Нет LinguistPro-карт в Anki → **«В Anki не найдено карточек LinguistPro (tag:lp_word). Сначала экспортируйте слова, затем
   учите их в Anki.»** + тост **«Нет карточек LinguistPro в Anki»**.
4. Провал → **«Не удалось синхронизировать: <msg>»** + тост **«Синхронизация из Anki не удалась»**.
5. Если `localMode='0'` → **«Синхронизация доступна только в локальном режиме (AnkiConnect).»** (кнопка «молчит» — проверь
   LOCAL_MODE, не дохлая кнопка).

**Ловушка (свежий импорт):** только что импортированная `.apkg`-карта имеет reps=0/type=0/ivl=0 → классифицируется как
**«ещё не учили <N>»** (materialize-guard: reviewed = reps>0 && (type≠0 || ivl>0)). Импортировал-и-сразу-Sync без учёбы →
«ещё не учили» это НЕ поломка. Сначала реально позанимайся.

### B3 — Наблюдаемые эффекты i+1 / coverage / retention

После успешного B2:
1. **Заново открой** текст со словами, которые учил в Anki:
   - они читаются как **«известно»**/mastered в **«🎯 Следующий для тебя»**;
   - рекомендации **«🎯 Следующий для тебя» / «🔥 Следующий вызов»** сдвигаются;
   - бейдж **«≈N% по твоим словам»** меняется.
2. **Retention:** суффикс **«· удержание <X>% (<R> повторов)»**. Если настоящего review-типа (type===1) ещё не было —
   честно **«—»** (null). Учитываются только review-типы (learning/relearning/cram/manual исключены; успех = ease≥3).
3. Per-text coverage / i+1-фронтир пересчитываются из `srs_cards` **при следующем открытии текста** (без живого re-render —
   нужно переоткрыть).

### B4 — Честность на телефоне

1. На телефоне открой модал «Anki».
2. **Ожидаемо:** AnkiConnect нет → **«Экспортировать»** и **«Синхронизировать из Anki»** не работают; подсказка и tooltip
   направляют на **«📦 Скачать .apkg»**. Документированное ограничение, не баг. Удалённый прод-пользователь B использовать
   НЕ может — B только там, где браузер и Anki на одной машине (localhost).

## G) Гарантии (тестируем как регрессии)

### G1 — .apkg-карта питает read-back (cross-transport identity по лемме)

**Repro:**
1. Desktop-предусловия (0.2: AnkiConnect + origin в webCorsOriginList + перезапуск).
2. Текст с несколькими ②-заметками → **🎯 SRS Тренажёр** → **«📦 Скачать .apkg (словарь)»** → `LinguistPro-Words-<дата>.apkg`.
   **Не используй** push — суть в `.apkg`-пути БЕЗ `lp_note_`.
3. В Anki: File → Import. Browse → кликни карту → **Tags**: есть **`lp_lemma_…`**, НЕТ ни одного **`lp_note_…`**.
4. Позанимайся этими картами (Good/Again), чтобы вышли из `new`.
5. В приложении → текст → **«Anki»** → **«Синхронизировать из Anki»**. Переоткрой текст. Нажми Sync ВТОРОЙ раз.

**PASS:** тост сообщает ненулевое **«обновлено <N>… удержание <X>%»**, НЕ «нет карточек / изменений нет». Слова читаются
как **«известно»**, i+1 сдвигается. Лемма-only карты дают `untagged:0`; mastery фанится на ВСЕ word_study-заметки той же
леммы. Второй Sync не дублирует счёт.

**Fail:** Sync говорит **«нет карточек LinguistPro»**/«изменений нет» после учёбы; слова остаются **«новое»**, i+1 не
двигается (лемма-only заметки попали в `untagged` и отброшены: AnkiIdentity не загружен / стороны по-разному хэшировали
ключ → `lemmaTagForBody(local) != lp_lemma_` тег карты).

### G2 — Rich global НЕ затирает rich-поля при повторном импорте

**Repro:**
1. Текст с ②-словами, у которых известное спряжение и у первого вхождения есть пример с аудио.
2. Текст → **«Anki»** → «Слова» → **«📦 Скачать .apkg»** (RICH модальная). Импортируй. Подтверди на карте **таблицу
   спряжения + пример** (+ аудио при «озвучке»).
3. **🎯 SRS Тренажёр** → **«📦 Скачать .apkg (словарь)»** (наблюдай **«⏳ <N>/<Total>…»**). Импортируй этот global-файл в
   **ТУ ЖЕ** коллекцию.
4. Anki показывает совпавшие как **«updated»**, не «added».
5. Переоткрой ранее-rich карты → **Conjugation, Example, Audio ВСЁ ЕЩЁ заполнены** (ничего не обнулилось).

## П) Платформенная матрица

| Возможность | Web (удалённый прод) | Desktop (браузер+Anki на одной машине) | Mobile (AnkiDroid/AnkiMobile) |
|---|---|---|---|
| **A1** Trainer global `.apkg` | ✅ (сборка в браузере) | ✅ | ✅ скачать на ПК → перенести → импорт |
| **A2** Модальный `.apkg` | ✅ (текст должен быть в Library) | ✅ | ✅ импорт файла |
| **B Push** («Экспортировать») | ❌ (нет localhost Anki) | ✅ LOCAL_MODE + webCorsOriginList | ❌ нет AnkiConnect |
| **B Read-back** («Синхронизировать из Anki») | ❌ | ✅ LOCAL_MODE-only | ❌ |
| **Health** («Проверить AnkiConnect») | ❌ | ✅ | ❌ |

## Т) Troubleshooting / ловушки ложного результата

1. **Stale Service Worker.** Старый SW отдаёт precache `anki-*.js` (грузятся БЕЗ `?v`) → тестируешь старый код → ложно
   «всё работает». Обязательно до тестов: SW = `v3.10.79-anki-g2-rich-export` И `window.AnkiIdentity` существует И
   `AnkiSrsExport.MODEL_NAME === 'LinguistPro Word v2'` И `AnkiIdentity.LEMMA_TAG_PREFIX === 'lp_lemma_'`.
2. **Audio force-cache ложный pass.** `fetch('/api/audio/<key>', {cache:'force-cache'})` может вернуть старый клип. «Звук
   играет» ≠ верно. Сверяй `asset_key → lp_<key>.mp3`, а не факт звука.
3. **Пустой профиль — ложный pass G2.** Без данных спряжения/примера обе поверхности тривиально «совпадают». Тестируй на
   заметке с реальной парадигмой + примером + аудио.
4. **Идемпотентность через ДРУГОЙ GUID.** Смена ключа леммы (POS/никуд/`pealim_id`) → GUID меняется → НОВАЯ карта (ложный
   fail «дубликат»). И наоборот: два настоящих омографа схлопываются в один `w:<word>#` (выглядит как дедуп, но это потеря).
   Сверь драйвер GUID до вердикта.
5. **Read-back на свежем импорте.** reps=0/type=0/ivl=0 → **«ещё не учили»**. Сначала позанимайся в Anki.
6. **CORS / mixed-content ложный FAIL.** **«Failed to fetch»** — это незаданный `webCorsOriginList` / Local-Network-Access,
   не поломка B. Добавь `location.origin` в `webCorsOriginList`, перезапусти Anki. С HTTPS-страницы на `127.0.0.1:8765`
   современный Chrome может ещё требовать разрешение Local Network Access.
7. **Retention double-count.** События `id 'anki:<reviewId>'` пишутся `INSERT OR IGNORE` — повтор Sync НЕ должен раздувать
   «<N> повторов». Растёт между sync → идемпотентность сломана. Старый AnkiConnect без `getReviewsOfCards` →
   `retentionAvailable=false` → **«—»** (честно).
8. **conflictKeptLocal — направление.** Conflict-guard оставляет локальное только если `localMs > ankiModMs`. Расхождение
   часов браузера/Anki может перевернуть сравнение. Проверяй намеренным порядком (ревью в приложении ПОСЛЕ ревью в Anki) —
   локальный interval не должен перезаписаться более старым Anki-состоянием (счётчик **«оставлено своё <N>»**).
9. **B в НЕ-локальном режиме.** `localMode='0'` → Sync/Push тихо отказывают: **«Синхронизация доступна только в локальном
   режиме (AnkiConnect).»** Подтверди LOCAL_MODE (default-on).
10. **«Fresh-only» неверно прочитан.** ☑ означает: карты с тем же ивритом в ДРУГИХ колодах остаются на месте. «push не
    тронул старую колоду» — документированное поведение, не баг.
11. **Телефон `{{tts}}` ложный fail.** Нет he-IL TTS-голоса → словесная лицевая молчит. Но встроенное `[sound:]`
    пример-аудио играет, `.apkg` корректен. Различай «нет device-TTS» и «нет export-audio».
12. **Метрика на «Оба типа».** Тост **«… (слов <W> · предложений <S> · аудио <M>)»** — если смотреть только итог, пропустишь
    `W=0` или `S=0`. Проверяй разбивку W/S.
13. **Front-mode на `.apkg` (не баг).** Дропдаун «Иврит на лицевой» читает только AnkiConnect SENTENCE-push, НЕ `.apkg`.
14. **Два имени файла.** GLOBAL = **`LinguistPro-Words-<дата>.apkg`** (с `-Words`); MODAL = **`LinguistPro-<дата>.apkg`**
    (без). Дата = `YYYY-MM-DD`. Перепутанный инфикс = сработала не та поверхность.
15. **`tag:lp_word` — единственный ключ read-back.** Карта без `lp_word` невидима для Sync (findNotes = `tag:lp_word`).
    Untagged/ручные/legacy намеренно скипаются (identity не угадывается по ивритской поверхности — homograph-proof).
