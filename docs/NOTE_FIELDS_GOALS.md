# Полевая система word_study-заметок — цели (acceptance-критерии)

> Канон целей для КАЖДОГО поля ②-заметки (`word_study`). Применяется при генерации
> бандла (`scripts/premium/build-notes-from-bundle.js`) и проверяется харнессом
> `scripts/premium/note-fields-audit.js` (`npm run audit:note-fields`). Роли R1–R5 —
> `docs/PROJECT_ROLES.md`. Инвариант владельца: **бескомпромиссное качество, без
> выдуманных форм; честный пустой > мусор.**

Поля `body_json`: `word`, `niqqud_variant`, `root`, `pos` (+ `part_of_speech`),
`binyan`, `meaning`. Доп. зашитая морфология верхнего уровня — `sentence_morph`
(Dicta-токены per-sentence, офлайн авто-заполнение POS/root в редакторе).

Источники истины: **Dicta** (`dictaMorph.js`) — `pos`/`root`/`niqqud`/`binyan`/
`prefix`/`stem` в контексте предложения; **Pealim** (резолвер + офлайн-словарь
`pealim-infl-v12`) — `meaning` (глосса) + парадигма.

## Цели по полям

| Поле | Формат | Содержательные (verb/noun/adj) | Служебные / имя собств. | Честный отказ |
|---|---|---|---|---|
| **word** | консонантный иврит, финальные буквы сохранены | обязательно | обязательно (как в тексте) | нет (required) |
| **niqqud_variant** | вокализованная форма (niqqud), без bidi-меток | из Dicta-токена | если есть в контексте | да (нет в Dicta) |
| **root** | триконсонантный корень (консонанты) | глагол → **обязателен**; сущ/прил → корень или честно пусто (заимств.) | **всегда пусто** | да (заимств./нет источника) |
| **pos** / **part_of_speech** | enum; ОБА ключа пишутся и **синхронны** | конкретный тип | конкретный тип (не "other") | пусто только если Dicta не дала сигнала |
| **binyan** | `paal/nifal/piel/pual/hifil/hufal/hitpael` | глагол → биньян или честно пусто | **всегда пусто** | да (Dicta не декодировала) |
| **meaning** | RU-глосса | присутствует (Pealim/словарь) | функц. глосса если есть; иначе пусто | да (нет в Pealim — loanword/slang) |

## Жёсткие инварианты R1 (gate — любое нарушение валит `audit:note-fields`)

- **R1-1** служебное слово (pronoun/preposition/conjunction/adverb/negation/
  interrogative/numeral/particle) → `root` **пусто**.
- **R1-2** не-глагол → `binyan` **пусто**; биньян — только у глаголов и только из
  валидного enum.
- **R1-3** глагол → `root` **не пусто**.
- **R1-4** имя собственное → `root` пусто, `pos="noun"` (`kind=propernoun`), `binyan` пусто.
- **KEYS** `pos` == `part_of_speech` (kmap читает `$.pos`, редактор — `part_of_speech`).
- **Никогда не выдумывать** root/binyan/pos/meaning — только Dicta/Pealim/офлайн-словарь.

## Мягкие цели (отчёт, не gate)

- `meaning` coverage — максимизировать из реальных источников; остаток (loanword/slang
  вне Pealim) — честный пустой.
- noun `root==lemma` («базовая форма», не триконсонантный корень) — осознанный R1-gap;
  настоящие именные корни — отдельная работа (true lemmatization, бэклог).
- `pos` пусто без морфологического сигнала Dicta (нет binyan, контекст-неоднозначно) —
  честный пустой; контекст-свободный POS из словаря НЕ ставим (это была бы догадка).

## Текущее состояние (бандл `Library/test-enriched.zip`, 8967 заметок)

word 100% · niqqud_variant 100% (99.8% вокализ.) · root 54.1% (0 R1-нарушений) ·
pos/part_of_speech 94.7% (desync 0) · binyan у глаголов 99.9% (0 R1-нарушений) ·
**meaning 96.2%** · R1 hard violations **0**.

Рычаги, поднявшие meaning 89.1%→96.2% (всё R1-честно, из реального Pealim-словаря):
1. **alias-fallback** — глосса по root/lemma из офлайн-словаря, когда live-резолв пуст.
2. **cell-reverse-lookup** — причастие, которое Dicta кросс-тегает как существительное
   (כותב, עובד, מבין), — это ячейка (AP-ms) парадигмы глагола; берём её глоссу, но
   **только при однозначности** после фильтра по биньяну (≥2 значения → пропуск).
3. **pos-инференция** — Dicta дала binyan без POS (מגהץ, מציירות) → `pos="verb"`
   (binyan — признак только глагола; сигнал Dicta, не догадка).

Остаток — честные пределы: loanword/slang вне Pealim (פנתר, דואט, דפוק), POS без сигнала
(формы היה, כל, צריך), именные корни (R1-gap).
