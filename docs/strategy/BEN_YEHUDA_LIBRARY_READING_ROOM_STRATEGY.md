> **Дата фиксации:** 2026-06-08
> **Статус:** ✅ **Owner decisions accepted** (решения владельца приняты)
> **Тип документа:** **canonical planning snapshot** — исходная зафиксированная
> стратегия. Это **НЕ** implementation plan и **НЕ** roadmap; это точка отсчёта,
> относительно которой ведётся последующий конкурентный аудит и gap-анализ

## Executive summary

LinguistPro обогащается полным корпусом текстов **Проекта Бен-Йехуда**
(https://benyehuda.org, https://github.com/projectbenyehuda) — 20 000+ канонических
произведений в общественном достоянии (Бялик, Черниховский, Менделе, Ахад-ха-Ам,
переводы мировой классики на иврит).

**Дыра рынка, которую мы занимаем:** никто не даёт *современный ивритский
литературный канон*, понятный на любом уровне, с native-аудио, глубокой
морфологией на тапе, **офлайн**. Beelinguapp — не градуирован и мал; LingQ — нет
аудио/никуда; Readlang — нет ивритской морфо-глубины; StoryHebrew — синтетика для
начинающих без канона; Sefaria — религиозный фокус. У нас уже построены **три
редких куска**: подлинный корпус, производственный движок (Dicta + офлайн-Pealim
9279 парадигм + перевод + GCP TTS + **i+1 frontier уже в проде**) и офлайн-first
премиум-UX.

**Центральное напряжение (честно):** Бен-Йехуда — литература высокого/архаичного
регистра, противоположность graded-контенту. Сырой дамп 20K = прекрасный архив, но
педагогический тупик. Поэтому победа = не «масштаб ради масштаба», а **«сделать
подлинный канон проходимым»** силами движка.

**Принятое решение:** Вариант **D — «Издательство + Читальный зал»** (синтез),
**отдельная страница-суб-бренд**, **два трека аудитории** (новички + intermediate+),
**гибридная модель производства** (кураторские полки = shipped офлайн-бандлы; полный
корпус = обогащение-при-открытии на BYOK).

## Принятые решения владельца (2026-06-08) — без сокращений

1. **Направление: D — синтез** («Издательство + Читальный зал»).
   Две поверхности, один движок: чистый «Читальный зал» спереди + «Студия»
   (нынешнее тяжёлое приложение) за кулисами как производственная мастерская.
2. **Размещение: отдельная страница / суб-бренд.**
   Новая `public/library.html`, обслуживается как статика подобно `teacher.html`;
   общий backend / OPFS / i18n. Чистое разделение Зал ↔ Студия — прямой ответ на
   формулировку владельца: «пользователь попадает просто в двуязычную библиотеку…
   без нагромождённого технического интерфейса».
3. **Аудитория: два трека параллельно.**
   «Доступная» полка (новички, on-ramp: детские стихи Бялика, притчи, простые
   отрывки Менделе, короткие тексты) + «Литературная» полка (intermediate+:
   полноценный канон с сильным скаффолдингом). Метки регистра / эпохи (R7/R8).
4. **Производство: гибрид.**
   Кураторские полки = заранее испечённые **shipped офлайн-бандлы** (стоимость
   ограничена, бесплатно пользователю, офлайн). «Глубокие стеллажи» всего корпуса =
   **обогащение-при-открытии** на **BYOK-ключах** пользователя.

> Эти четыре решения — инвариант. Последующий аудит/бэклог не пересматривает их, а
> уточняет, *как* их реализовать и в каком порядке.

## Факты, на которых стоит стратегия

### Корпус Бен-Йехуда
- **20 000+ произведений** в общественном достоянии (Israeli law: 70 лет после
  смерти автора), 300+ авторов; проза, поэзия, эссе, справочники и **переводы
  мировой классики на иврит** — канон современного ивритоязычного письма.
- Форматы: `txt/` (с никудом), `txt_stripped/` (без никуда), `html/`; структура —
  **один каталог на автора**; индекс `pseudocatalogue.csv` (название, автор, жанр,
  путь). Полный дамп — в Releases (последний релиз — март 2026), есть зеркало на
  HuggingFace и бесплатный API на benyehuda.org.
- **Лицензия:** свободное использование; просьба — **цитировать источник**
  (атрибуция per work → R6).

### Готовый к переиспользованию движок (ничего не строить с нуля)
- **Текст→двуязычие:** `db/premium/pipeline.js#translateTable()` (сегментация +
  никуд-gateway + перевод madlad/GCP/free + двойная транслитерация); трёхуровневый
  кэш (`translation_doc_cache`/`segment_cache`/`overrides`). BYOK.
- **Аудио:** `server.js#ensureAudioAsset()` (стабильный `asset_key`, привязка к
  предложениям), `POST /api/tts` (GCP TTS, BYOK), кэш `audio-cache/*.mp3`.
- **Данные:** `texts` + `sentences`; `db/libraryRepo.js#createTextWithSentences()`.
- **Bulk import:** ZIP-бандл schema v2 (`/api/library/import/bundle`), режимы
  skip/asNew, авто-бэкап, дедуп аудио по SHA256; `scripts/premium/build-notes-from-bundle.js`.
- **Читалка:** `renderTable()` (he|niqqud|translit|ru, ▶/📝), `v3LibraryOpenText()`,
  `v3MapSentenceApiRowToUiRow()`.
- **Педагогические активы УЖЕ В ПРОДЕ:** известные слова + **i+1 frontier**
  (`getTextLearningCoverage`), частотность корней (`rankRoots`), **конкорданс по
  всему корпусу** (`public/js/crosstext.js`), офлайн-словарь Pealim (9279 парадигм),
  Dicta-морфология, OPFS офлайн-first, Anki export + двусторонний sync.
- **Отдельные страницы** обслуживаются как статика → новая страница тривиальна;
  i18n ru/en/he через `data-i18n`/`applyI18n`.

## Новые роли (профессиональный создатель учебных библиотек)

Дополняют R1–R5 (`docs/PROJECT_ROLES.md`).

### R6 — Куратор-библиотекарь / collection developer
**Мандат:** библиотека как система, а не дамп.
- Политика комплектования, левелинг, схема метаданных (автор/жанр/эпоха/сложность/
  темы), discoverability (полки, фасеты, поиск), дедуп, weeding, **атрибуция
  Бен-Йехуда per work**.
- 🚩 **Красные флаги:** дамп вместо библиотеки; отсутствие левелинга; сломанный
  поиск/навигация; отсутствие цитирования источника; метаданные, которые потом
  больно мигрировать.

### R7 — Литературовед-гебраист / редактор
**Мандат:** суждение о каноне и честность с регистром.
- Что и в каком порядке читать; значимость авторов/произведений; культурно-
  исторический контекст; редакторские интро; **честная работа с архаикой/регистром**.
- 🚩 **Красные флаги:** свести канон к спискам слов; мис-контекстуализация; подача
  тяжёлой поэзии как «для начинающих»; игнор разницы литературный ≠ разговорный иврит.

### R8 — Дизайнер обучающего контента / graded-reading
**Мандат:** проходимость и предотвращение тупиков (углубляет R2 для библиотеки).
- Методология грейдинга сложности; дизайн скаффолдинга (никуд/транслит/глосса/
  параллель — **с угасанием**); on-ramp; пейсинг; «что читать дальше».
- 🚩 **Красные флаги:** скаффолдинг, который не убирается; ramps без грейдинга;
  тупики понимания; пугающая техническая терминология сложности.

## Финальный подход D — «Издательство + Читальный зал»

**Две поверхности, один движок.**

### Читальный зал (Reading Room)
Новая `public/library.html` (+ `public/js/library-ui.js`): чистая двуязычная
читалка с аудио. На входе — две полки (Доступная / Литературная), консоль
скаффолдинга, «глубокие стеллажи» за курацией. Переиспользует `renderTable`,
`v3MapSentenceApiRowToUiRow`, OPFS `local-db.js`, аудио-слой, i18n (namespace
`room.*`), bundle-import. SW — отдельный лёгкий precache (без морфо/граф/квиз-чанков).
**Скрыты** морфо-редактор, квизы, граф-редактор, Anki/SRS, bulk-import, редактирование.

### Студия (Studio)
Существующее `index.html`: мастерская, где владелец/куратор гонит сырые тексты
Бен-Йехуда через `build-notes-from-bundle.js` → ZIP-бандлы → импорт в общий OPFS,
видимый в Зале. Все «тяжёлые» функции остаются здесь.

### Фазировка
1. **Фаза 1 — Зал + две полки (B, гибрид-shipped).**
   Конвейер ингестии: парс `pseudocatalogue.csv` + `txt/` (с никудом) → сегментация
   (`pipeline.js`) → перевод+транслит+аудио → `createTextWithSentences` → ZIP v2.
   Курация ~50–150 произведений в два трека; метки `level`/`tags`/`era`/`register`;
   провенанс+атрибуция per work. Зал: полки → текст → `renderTable` (he|niqqud|
   translit|ru, ▶). @380px RTL скриншот (критерий R4).
2. **Фаза 2 — Скаффолдинг + i+1 (C).**
   Консоль: никуд (полный→частичный→нет), транслит вкл/выкл, тап-глосса, раскрытие
   перевода (скрыт→предложение→всё), скорость. Собственный оценщик сложности (Dicta
   type/token + `rankRoots` частотность + длина/клаузы). Персональные рекомендации
   через `getTextLearningCoverage` (i+1 уже в проде).
3. **Фаза 3 — Глубокие стеллажи (A, BYOK).**
   Каталог всего корпуса из `pseudocatalogue.csv`; поиск по автору/жанру;
   обогащение-при-открытии на ключах пользователя; honest empty/online-states;
   конкорданс (`crosstext.js`) по всему канону. Метка «машинно-ассистировано».
4. **Фаза 4 — Кураторские маршруты.**
   Путешествия по авторам, тематические полки, шаринг полок через ZIP, опц. мостик
   «открыть в Студии».

## Ультимативные преимущества (моат)
1. Единственный дом **современного ивритского литературного канона** как
   learner-grade двуязычной аудио-библиотеки (подлинность+масштаб Бен-Йехуда).
2. **Персонализированная сложность / i+1 на подлинной литературе** — уже зашитый
   frontier-движок; для иврита этого нет ни у кого.
3. **Глубокая морфология на тапе внутри чтения** — Dicta + офлайн-Pealim (9279
   парадигм) — уровня Pealim, офлайн.
4. **Регулируемый скаффолдинг** (никуд/транслит/глосса/параллель) — один подлинный
   текст служит A2→C2 (у StoryHebrew только fade никуда, не весь стек на каноне).
5. **Офлайн-first премиум** — полка читается+звучит офлайн (OPFS-бандлы).
6. **Провенанс и честность** (R1/R4) — настоящее аудио/формы, цитируемый
   public-domain источник; доверие как фича.
7. **Двусторонний пайплайн Студия→Зал** — кураторские и community-полки через ZIP.
8. **Конкорданс по всему канону** — употребление слова через всю библиотеку.

## Риски / честные ограничения
- Архаичный/литературный регистр ≠ современный разговорный → метить «литературный/
  эпоха»; отдельные «доступные» полки (R8/R7).
- MT-перевод поэзии слаб → кураторская полка = вычитанный перевод; глубокие стеллажи
  помечены «машинно-ассистировано» (провенанс R1/R5).
- TTS на поэзии/архаике: просодия неидеальна, никуд помогает → метить провенанс;
  разрешить импорт человеческого аудио.
- Стоимость на масштабе → граница «precompute curated / on-demand BYOK» (R5).
- Атрибуция Бен-Йехуда per work (R6).

## Критерии проверки / валидации
- Пилот-полка 10–20 текстов через `build-notes-from-bundle.js` → ZIP-import →
  `renderTable` офлайн @380px RTL (оба трека).
- Оценщик сложности: sanity на размеченной выборке vs частотность/i+1.
- Провенанс/атрибуция Бен-Йехуда в UI; honest empty-states «глубоких стеллажей».
- Перед UI-коммитом: скриншот @380px (норма CLAUDE.md); SW — отдельный precache
  Зала, не бампать основной `CACHE_VERSION` без нужды.
